## 1. 背景知识：程序运行的内存模型

### 1.1 虚拟内存布局

一个 x86-64 程序运行时的虚拟内存布局如下：

```
┌─────────────────────┐ 高地址 (0x7FFF...)
│     栈 (Stack)       │ ← %rsp 指向栈顶
│   (向下增长)          │
├─────────────────────┤
│         ↓            │
│      (空闲)           │
│         ↑            │
├─────────────────────┤
│     堆 (Heap)        │ ← malloc 分配
├─────────────────────┤
│   数据段 (Data)       │ ← 全局变量、静态变量
├─────────────────────┤
│   代码段 (Text)       │ ← 可执行指令
└─────────────────────┘ 低地址 (0x0000...)
```

💡 **关键点**：
- 栈在 x86-64 中**向低地址增长**（"倒着画"）
- `%rsp` 始终指向当前栈顶（最低地址处）
- 每个函数调用时会在栈上创建自己的 **栈帧 (Stack Frame)**

### 1.2 为什么需要栈？

过程调用需要解决三个问题：
1. **传递控制**：从调用者跳转到被调用者，完成后返回
2. **传递数据**：参数和返回值
3. **管理局部存储**：局部变量的空间分配与释放

栈的 LIFO（后进先出）特性完美匹配函数调用的嵌套结构：
```
main() → foo() → bar() → 返回 → foo() → 返回 → main()
```

## 2. x86-64 程序视图与寄存器

### 2.1 通用寄存器及用途

x86-64 有 16 个 64 位通用寄存器：

| 寄存器        | 用途           | 调用约定                    |
| ------------- | -------------- | --------------------------- |
| `%rax`        | 返回值         | 调用者保存 (Caller-saved)   |
| `%rdi`        | 第1个参数      | 调用者保存                  |
| `%rsi`        | 第2个参数      | 调用者保存                  |
| `%rdx`        | 第3个参数      | 调用者保存                  |
| `%rcx`        | 第4个参数      | 调用者保存                  |
| `%r8`         | 第5个参数      | 调用者保存                  |
| `%r9`         | 第6个参数      | 调用者保存                  |
| `%r10`-`%r11` | 临时使用       | 调用者保存                  |
| `%rbx`        | 临时/基址      | 被调用者保存 (Callee-saved) |
| `%rbp`        | 帧指针（可选） | 被调用者保存                |
| `%r12`-`%r15` | 临时使用       | 被调用者保存                |
| `%rsp`        | 栈指针         | 被调用者保存（特殊）        |

### 2.2 调用约定速查表

| 概念                            | 含义                                                    |
| ------------------------------- | ------------------------------------------------------- |
| **调用者保存 (Caller-saved)**   | 调用者负责保存和恢复这些寄存器的值                      |
| **被调用者保存 (Callee-saved)** | 被调用者负责保存和恢复这些寄存器的值                    |
| **参数寄存器**                  | `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`, `%r9`（最多6个） |
| **返回值寄存器**                | `%rax`                                                  |

## 3. 栈操作：push 和 pop

### 3.1 `pushq` 指令

**语法**：`pushq Src`

**操作**：
1. `%rsp = %rsp - 8`（栈向下增长8字节）
2. `M[%rsp] = Src`（将Src的值写入新的栈顶）

**示例**：
```asm
pushq %rbp        # 将 %rbp 的值压栈
```

**等价于**：
```asm
subq $8, %rsp     # 栈指针下移8字节
movq %rbp, (%rsp) # 将 %rbp 存入栈顶
```

### 3.2 `popq` 指令

**语法**：`popq Dest`

**操作**：
1. `Dest = M[%rsp]`（从栈顶读取值到Dest）
2. `%rsp = %rsp + 8`（栈指针上移8字节）

**示例**：
```asm
popq %rdx         # 从栈顶弹出值到 %rdx
```

**等价于**：
```asm
movq (%rsp), %rdx # 从栈顶读取到 %rdx
addq $8, %rsp     # 栈指针上移8字节
```

⚠️ **重要**：`popq` 实际上**不会清除**内存中的旧值，只是移动栈指针。旧数据仍然留在内存中，但不再属于栈。

### 3.3 栈操作示例

```
初始状态：
%rsp = 0x1000

pushq %rbp:
  1. %rsp = 0x0FF8
  2. M[0x0FF8] = %rbp

popq %rdx:
  1. %rdx = M[0x0FF8]
  2. %rsp = 0x1000
```

❌ **常见错误**：
```c
// 错误：忘记恢复栈指针
void bad_function() {
    // 压栈后没有对应的弹出
}
// 正确：push 和 pop 必须成对出现
```

✅ **正确做法**：
```asm
pushq %rbp        # 保存
# ... 函数代码 ...
popq %rbp         # 恢复
ret
```

## 4. 过程调用的核心机制

### 4.1 `call` 指令

**语法**：`call 目标地址`

**操作**：
1. `%rsp = %rsp - 8`（栈下移）
2. `M[%rsp] = 返回地址`（call的下一条指令地址）
3. 跳转到目标地址

### 4.2 `ret` 指令

**语法**：`ret`

**操作**：
1. 从 `M[%rsp]` 读取返回地址
2. `%rsp = %rsp + 8`
3. 跳转到返回地址

### 4.3 调用流程图

```
调用者 (Caller)              被调用者 (Callee)
     │                              │
     │  call 函数地址               │
     ├──────────────────────────────►│
     │  (压入返回地址)               │
     │                              │
     │                         执行函数体
     │                              │
     │                          ret 返回
     │◄──────────────────────────────┤
     │  (弹出返回地址)               │
     │                              │
     ▼                              ▼
```

### 4.4 完整的调用示例

```c
// C代码
void multstore(long x, long y, long *dest) {
    long t = mult2(x, y);
    *dest = t;
}

long mult2(long a, long b) {
    return a * b;
}
```

对应的汇编：
```asm
0000000000400540 <multstore>:
   400540: push %rbx              # 保存 %rbx（被调用者保存）
   400541: mov  %rdx, %rbx        # 保存 dest 到 %rbx
   400544: call 400550 <mult2>    # 调用 mult2(x, y)
   400549: mov  %rax, (%rbx)      # 将返回值存入 dest
   40054c: pop  %rbx              # 恢复 %rbx
   40054d: ret                    # 返回

0000000000400550 <mult2>:
   400550: mov  %rdi, %rax        # a → %rax
   400553: imul %rsi, %rax        # a * b → %rax
   400557: ret                    # 返回
```

📝 **执行过程**：
1. `call 400550`：将返回地址 `0x400549` 压栈，跳转到 `mult2`
2. `mult2` 执行乘法，结果在 `%rax`
3. `ret`：弹出返回地址 `0x400549`，跳转回去
4. 继续执行 `mov %rax, (%rbx)`

## 5. 参数传递与返回值

### 5.1 参数传递规则

**前6个参数**：按顺序存入寄存器

| 参数序号 | 寄存器 |
| -------- | ------ |
| 1        | `%rdi` |
| 2        | `%rsi` |
| 3        | `%rdx` |
| 4        | `%rcx` |
| 5        | `%r8`  |
| 6        | `%r9`  |

**第7个及以后**：压入栈中（从右到左）

### 5.2 返回值

返回值始终放在 `%rax` 寄存器中。

### 5.3 完整示例

```c
// 有6个参数
long add6(long a, long b, long c, long d, long e, long f) {
    return a + b + c + d + e + f;
}
```

```asm
add6:
    # a=%rdi, b=%rsi, c=%rdx, d=%rcx, e=%r8, f=%r9
    addq %rsi, %rdi    # a += b
    addq %rdx, %rdi    # a += c
    addq %rcx, %rdi    # a += d
    addq %r8, %rdi     # a += e
    addq %r9, %rdi     # a += f
    movq %rdi, %rax    # 返回值 → %rax
    ret
```

### 5.4 超过6个参数的示例

```c
long add10(long a0, long a1, long a2, long a3, long a4,
           long a5, long a6, long a7, long a8, long a9) {
    return a0 + a1 + a2 + a3 + a4 +
           a5 + a6 + a7 + a8 + a9;
}
```

参数分布：
- `a0`-`a4` → `%rdi`, `%rsi`, `%rdx`, `%rcx`, `%r8`
- `a5` → `%r9`
- `a6`-`a9` → 栈上（从 `(%rsp)+8` 开始）

## 6. 栈帧管理

### 6.1 什么是栈帧？

每个函数调用在栈上占据一块连续的内存区域，称为**栈帧 (Stack Frame)**。

```
                    ┌─────────────────┐ 高地址
                    │  调用者栈帧      │
                    │  (Caller Frame)  │
                    ├─────────────────┤
                    │  返回地址        │ ← 由 call 压入
                    ├─────────────────┤
    当前栈帧 →      │  保存的 %rbp     │ ← 可选
     (Callee)       ├─────────────────┤
                    │  局部变量        │
                    │  临时存储        │
                    ├─────────────────┤
                    │  ...            │
                    └─────────────────┘ 低地址 ← %rsp
```

### 6.2 栈帧的创建和销毁

**创建栈帧**（函数开头）：
```asm
pushq %rbp          # 保存旧的帧指针
movq  %rsp, %rbp    # 设置新的帧指针
subq  $N, %rsp      # 为局部变量分配 N 字节空间
```

**销毁栈帧**（函数返回前）：
```asm
movq  %rbp, %rsp    # 恢复栈指针
popq  %rbp          # 恢复旧的帧指针
ret                 # 返回
```

### 6.3 不需要栈帧的情况

如果函数：
- 参数不超过6个
- 没有局部变量
- 不需要保存寄存器

则不需要分配栈帧。例如 `mult2` 就没有栈帧。

```asm
mult2:
    mov  %rdi, %rax
    imul %rsi, %rax
    ret
```

## 7. 调用约定：保存与恢复寄存器

### 7.1 为什么需要约定？

```
void caller() {
    long x = 100;
    callee();        // callee 可能会修改 %rax
    // 但这里我们需要 x 的值
}
```

如果 `callee` 修改了某个寄存器，而 `caller` 还期望它的值保持不变，就会出问题。

### 7.2 两类寄存器的约定

| 类型         | 寄存器                               | 谁负责保存             |
| ------------ | ------------------------------------ | ---------------------- |
| 调用者保存   | `%rax`, `%rdi`-`%r9`, `%r10`, `%r11` | 调用者                 |
| 被调用者保存 | `%rbx`, `%rbp`, `%r12`-`%r15`        | 被调用者               |
| 特殊         | `%rsp`                               | 被调用者（返回时恢复） |

### 7.3 调用者保存示例

```c
long caller_save_example(long x) {
    // 需要使用 %rdi 调用 incr，但 x 在 %rdi 中
    // 必须保存 x 到别处
    long y = incr(&x, 3000);
    return x + y;
}
```

```asm
caller_save_example:
    pushq %rdi           # 保存 x 到栈上 (调用者保存)
    movq  %rsp, %rdi     # 将 &x 作为参数
    movq  $3000, %rsi    # val=3000
    call  incr
    popq  %rdx           # 恢复 x 到 %rdx
    addq  %rax, %rdx     # x + incr的结果
    movq  %rdx, %rax     # 返回
    ret
```

### 7.4 被调用者保存示例

```c
long callee_save_example(long x) {
    // 需要保存 x 到调用结束后使用
    long result = x + 1;
    // ... 其他操作
    return result;
}
```

```asm
callee_save_example:
    pushq %rbx           # 保存 %rbx（被调用者保存）
    movq  %rdi, %rbx     # 保存 x 到 %rbx
    addq  $1, %rbx       # x + 1
    # ... 其他代码可以使用 %rbx
    movq  %rbx, %rax     # 返回值
    popq  %rbx           # 恢复 %rbx
    ret
```

## 8. 递归函数实现

### 8.1 递归函数的栈帧

递归调用时，每次调用都会创建新的栈帧，相互独立。

### 8.2 完整示例：递归计算 popcount

```c
/* 递归计算二进制中1的个数 */
long pcount_r(unsigned long x) {
    if (x == 0)
        return 0;
    else
        return (x & 1) + pcount_r(x >> 1);
}
```

汇编实现：
```asm
pcount_r:
    movl  $0, %eax        # 默认返回值0
    testq %rdi, %rdi      # 测试 x == 0
    je    .L6             # 如果是0，返回

    pushq %rbx            # 保存 %rbx（被调用者保存）
    movq  %rdi, %rbx      # 保存原始 x 到 %rbx
    andl  $1, %ebx        # rbx = x & 1 (最低位)
    shrq  %rdi            # x = x >> 1 (参数准备)
    call  pcount_r        # 递归调用
    addq  %rbx, %rax      # 结果 += (x & 1)
    popq  %rbx            # 恢复 %rbx
.L6:
    rep; ret              # 返回
```

📝 **执行过程**（假设 `pcount_r(5)`，二进制 `101`）：

1. `pcount_r(5)`：`x=5`，保存 `x&1=1`，调用 `pcount_r(2)`
2. `pcount_r(2)`：`x=2`，保存 `x&1=0`，调用 `pcount_r(1)`
3. `pcount_r(1)`：`x=1`，保存 `x&1=1`，调用 `pcount_r(0)`
4. `pcount_r(0)`：返回 0
5. 回溯：`1+0` → `0+1` → `1+1` = 2 ✓

### 8.3 递归的关键点

✅ **正确做法**：
- 每次递归调用前保存必要的状态（使用被调用者保存寄存器）
- 使用 `%rbx` 等保存中间值
- 调用后从 `%rax` 获取返回值

❌ **常见错误**：
```asm
# 错误：没有保存中间值，直接被覆盖
pcount_r_wrong:
    # 错误！%rdi 被修改后无法恢复
    andl  $1, %edi
    # 这里丢失了原始 x 的值
```

## 9. 完整示例：从C到汇编

### 9.1 示例代码

```c
// file: procedure_example.c
#include <stdio.h>

long incr(long *p, long val) {
    long x = *p;      // 读取 p 指向的值
    long y = x + val; // 加 val
    *p = y;           // 写回
    return x;         // 返回旧值
}

long call_incr() {
    long v1 = 15213;
    long v2 = incr(&v1, 3000);
    return v1 + v2;
}

int main() {
    long result = call_incr();
    printf("Result: %ld\n", result);
    return 0;
}
```

### 9.2 编译查看汇编

```bash
gcc -O0 -S procedure_example.c
cat procedure_example.s
```

### 9.3 汇编分析

```asm
incr:
    # p 在 %rdi, val 在 %rsi
    movq (%rdi), %rax    # x = *p
    addq %rax, %rsi      # y = x + val
    movq %rsi, (%rdi)    # *p = y
    ret                  # 返回 x (%rax)

call_incr:
    pushq %rbp           # 保存帧指针
    movq  %rsp, %rbp     # 设置帧指针
    subq  $16, %rsp      # 分配16字节局部空间

    movq  $15213, -8(%rbp)  # v1 = 15213

    leaq  -8(%rbp), %rdi    # &v1 → %rdi
    movq  $3000, %rsi       # val = 3000
    call  incr              # 调用 incr(&v1, 3000)

    movq  -8(%rbp), %rdx    # v1
    addq  %rax, %rdx        # v1 + v2
    movq  %rdx, %rax        # 返回值

    leave                  # 恢复栈（等价于 mov %rbp,%rsp; pop %rbp）
    ret
```

### 9.4 内存布局变化

**调用 `incr` 之前**：
```
高地址
┌──────────────┐
│  返回地址     │ ← %rsp 指向这里
├──────────────┤
│              │
│  局部变量     │
│  保存的%rbp   │
└──────────────┘
低地址
```

**`incr` 执行中**：
```
高地址
┌──────────────┐
│  返回地址     │
├──────────────┤
│  v1 = 15213  │ ← p 指向这里
├──────────────┤
│              │
└──────────────┘
低地址 ← %rsp
```

## 10. 速查卡片

### 10.1 核心指令

| 指令        | 操作     | 等价代码                         |
| ----------- | -------- | -------------------------------- |
| `pushq Src` | 压栈     | `subq $8,%rsp; movq Src,(%rsp)`  |
| `popq Dest` | 弹栈     | `movq (%rsp),Dest; addq $8,%rsp` |
| `call 目标` | 调用函数 | `pushq 下一条地址; jmp 目标`     |
| `ret`       | 返回     | `popq 地址; jmp 地址`            |

### 10.2 寄存器用途

| 寄存器 | 用途   | 保存者   |
| ------ | ------ | -------- |
| `%rax` | 返回值 | 调用者   |
| `%rdi` | 参数1  | 调用者   |
| `%rsi` | 参数2  | 调用者   |
| `%rdx` | 参数3  | 调用者   |
| `%rcx` | 参数4  | 调用者   |
| `%r8`  | 参数5  | 调用者   |
| `%r9`  | 参数6  | 调用者   |
| `%rbx` | 通用   | 被调用者 |
| `%rbp` | 帧指针 | 被调用者 |
| `%rsp` | 栈指针 | 特殊     |

### 10.3 标准函数序言/结语

**序言**（函数开头）：
```asm
pushq %rbp
movq  %rsp, %rbp
subq  $N, %rsp      # N = 局部变量所需空间
```

**结语**（函数返回前）：
```asm
movq  %rbp, %rsp
popq  %rbp
ret
```

或使用 `leave` 指令：
```asm
leave   # 等价于 movq %rbp,%rsp; popq %rbp
ret
```

### 10.4 参数传递速查

```
C:  void func(a, b, c, d, e, f, g)
         │   │   │   │   │   │   │
x86-64:  rdi rsi rdx rcx r8  r9  (%rsp)+8
```

## 11. 核心要点总结

### ✅ 必须掌握的概念

- [ ] **栈向下增长**：`%rsp` 减小表示压栈，增大表示弹栈
- [ ] **call/ret 配对**：`call` 压入返回地址，`ret` 弹出
- [ ] **前6个参数在寄存器**：`%rdi`-`%r9`，其余在栈上
- [ ] **返回值在 `%rax`**：所有函数都通过 `%rax` 返回
- [ ] **调用者保存 vs 被调用者保存**：需要约定来避免寄存器冲突
- [ ] **每个函数有独立栈帧**：递归和普通调用都一样

### 💡 实用技巧

1. **调试时检查栈**：`info registers` 查看 `%rsp`，`x/16xg $rsp` 查看栈内容
2. **不要忘记保存 `%rbx`**：如果要用，先 `pushq %rbx`，返回前 `popq %rbx`
3. **局部变量足够小**：不需要栈帧时就不分配，代码更高效
4. **画图理解**：栈的变化画出来更容易理解

### ❌ 常见错误

| 错误                         | 后果                     | 正确做法                     |
| ---------------------------- | ------------------------ | ---------------------------- |
| push/pop 不配对              | 栈指针错乱，返回地址错误 | 确保每个 push 都有对应的 pop |
| 忘记保存调用者保存寄存器     | 调用函数后值丢失         | 调用前保存，调用后恢复       |
| 使用被调用者保存寄存器不恢复 | 调用者期望的值被破坏     | 使用前保存，使用后恢复       |
| 超过6个参数传递错误          | 参数错位                 | 第7个开始放在栈上            |
| 递归中覆盖参数寄存器         | 递归调用后参数丢失       | 使用 `%rbx` 等保存中间值     |

### ⚠️ 重要注意事项

1. **`popq` 不会清除内存**：只是移动指针，旧数据还在
2. **栈空间按16字节对齐**：特别是 `call` 指令前后
3. **`%rbp` 是可选的**：现代编译器可能使用 `%rsp` 直接访问
4. **递归没有特殊处理**：和普通调用一样，靠栈帧保证独立性

### 📈 时间复杂度

- **`push`/`pop`**：O(1)
- **`call`/`ret`**：O(1)
- **函数调用开销**：通常 10-20 个时钟周期
- **栈帧分配**：O(1)（通常是 `subq` 指令）

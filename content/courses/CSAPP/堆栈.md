# 内存管理深度解析——栈与堆

## 课程概述

围绕C语言中**内存管理的核心概念**展开，重点讲解栈(Stack)与堆(Heap)的区别、使用方法以及动态内存管理的实践技巧。

---

## 一、内存管理基础

### 1.1 为什么需要理解内存管理？

- **高效数据传递**：通过指针避免大块数据的副本拷贝
- **手动内存控制**：堆内存允许程序员精确控制内存生命周期
- **安全编程**：理解`use-after-free`等常见漏洞的成因

### 1.2 进程内存布局

一个C程序运行时，其内存空间被划分为多个段(Segments)：

```
       高地址
    +-------------+
    |    栈段     |  ← 向下增长 (函数调用、局部变量)
    |   (Stack)   |
    +-------------+
    |      ↓      |
    |             |
    |      ↑      |
    +-------------+
    |    堆段     |  ← 向上增长 (动态分配)
    |    (Heap)   |
    +-------------+
    |   数据段    |  (全局变量、静态变量、常量)
    |    (Data)   |
    +-------------+
    |   代码段    |  (程序指令)
    |   (Text)    |
    +-------------+
       低地址
```

**各段说明：**

- **代码段(Text)**：存放程序的机器指令，只读
- **数据段(Data)**：存放全局变量、静态变量、字符串常量
- **堆(Heap)**：动态分配的内存区域，程序员手动管理
- **栈(Stack)**：存储函数调用帧、局部变量、参数，自动管理

> **关键特性**：栈向下增长，堆向上增长。两者相向而行，中间的空闲区域供动态扩展使用。

## 二、指针与数组深度解析

### 2.1 指针的本质

指针是一个**存储内存地址的变量**。在64位系统上，指针占用8字节。

```c
int x = 42;
int *ptr = &x;  // ptr 存储了变量 x 的地址

// 内存示意图：
地址:     0x7ffeeb0c  0x7ffeeb08  0x7ffeeb04
内容:    [   42    ]  [  0x7ffeeb0c ]  [   ...   ]
         变量 x       指针 ptr        其他
```

**核心操作：**

- `&` 取地址运算符：获取变量的内存地址
- `*` 解引用运算符：访问指针指向的内存位置的值

### 2.2 数组与指针的关系

```c
int arr[5] = {10, 20, 30, 40, 50};
int *ptr = arr;    // arr 退化为指向第一个元素的指针
// 等价写法：int *ptr = &arr[0];
```

**关键知识点：**

| 特性     | 数组                 | 指针                |
| -------- | -------------------- | ------------------- |
| 内存分配 | 编译时确定大小       | 存储一个地址(8字节) |
| sizeof   | 返回整个数组的字节数 | 返回8字节           |
| 赋值     | 不能作为左值重新赋值 | 可以指向不同地址    |
| 作为参数 | 退化为指针           | 保持指针性质        |

**数组"退化"的含义：**

```c
void func(int arr[]) {
    // 此时 arr 实际上是一个 int* 指针
    printf("%zu\n", sizeof(arr));  // 输出 8，而非数组大小
}
```

### 2.3 指针与字符串

```c
char *myStr = " hi";      // myStr 指向数据段中的字符串常量
skipSpaces(&myStr);       // 传入指针的地址，允许修改指针本身
```

**双重指针的应用：**

```c
void skipSpaces(char **strPtr) {
    // 跳过前导空格
    while (**strPtr == ' ') {
        (*strPtr)++;      // 修改原指针的指向
    }
}
```

> **易错点**：字符串常量存储在只读数据段，修改其内容会导致未定义行为。应使用数组或动态分配的可修改字符串。

## 三、栈(Stack)——自动管理的内存

### 3.1 栈的工作原理

栈采用**后进先出(LIFO)**结构。每次函数调用会**压入(Push)**一个栈帧，函数返回时**弹出(Pop)**该帧。

```c
void func2() {
    int d = 0;          // 栈帧2中的局部变量
}

void func1() {
    int c = 99;         // 栈帧1中的局部变量
    func2();            // 调用 func2，压入新帧
}

int main() {
    int a = 42;         // main 栈帧中的变量
    int b = 17;
    func1();            // 调用 func1，压入新帧
    return 0;
}
```

**栈帧的内存布局：**

```
       高地址
    +-------------+  ← 调用前
    | main 栈帧   |
    |  - a (42)   |
    |  - b (17)   |
    +-------------+  ← func1() 调用时
    | func1 栈帧  |
    |  - c (99)   |
    +-------------+  ← func2() 调用时
    | func2 栈帧  |
    |  - d (0)    |
    +-------------+  ← 当前栈顶
       低地址
```

### 3.2 栈的局限性与常见错误

**局限性：**

1. 栈空间有限（通常8MB），大数组可能导致**栈溢出**
2. 函数返回后，栈帧中的局部变量**不再有效**，其内存可能被后续调用覆盖

**典型错误——返回局部变量的地址：**

```c
char *create_string_bad(char ch, int num) {
    char new_str[num + 1];      // 在栈上分配
    for (int i = 0; i < num; i++) {
        new_str[i] = ch;
    }
    new_str[num] = '\0';
    return new_str;              // ❌ 错误！返回指向栈内存的指针
}

int main() {
    char *str = create_string_bad('a', 4);
    printf("%s\n", str);         // 未定义行为：可能输出乱码或崩溃
    return 0;
}
```

### 3.3 递归与栈

递归函数每次调用都会创建新的栈帧：

```c
int factorial(int n) {
    if (n == 1) {
        return 1;
    } else {
        return n * factorial(n - 1);
    }
}

// factorial(4) 的调用栈：
// factorial(4) → factorial(3) → factorial(2) → factorial(1)
// 然后逐层返回：1 → 2 → 6 → 24
```

> **注意**：过深的递归会导致**栈溢出**。对于大规模递归，考虑改用迭代或堆内存。

## 四、堆(Heap)——手动管理的内存

### 4.1 为什么需要堆？

- 数据需要在函数调用结束后**持续存在**
- 运行时才能确定所需内存**大小**
- 需要动态**调整**已分配的内存大小

### 4.2 堆内存管理三步骤

**工作流程：**

```
1. 分配 (Allocate)  →  malloc/calloc/strdup/realloc
2. 断言 (Assert)    →  检查返回值是否为 NULL
3. 释放 (Free)      →  free()
```

### 4.3 核心分配函数

#### malloc——原始内存分配

```c
void *malloc(size_t size);

// 示例：分配存储10个int的空间
int *arr = malloc(10 * sizeof(int));
if (arr == NULL) {
    // 处理内存分配失败
}
```

**特点：**
- 返回 `void*`，可隐式转换为任何指针类型
- **不初始化内存**，内容为随机值
- 参数为**字节数**，不是元素个数

#### calloc——清零分配

```c
void *calloc(size_t nmemb, size_t size);

// 示例：分配并清零10个int
int *arr = calloc(10, sizeof(int));  // 所有元素初始化为0
```

**特点：**
- 内存被**初始化为0**
- 参数为`(元素个数, 每个元素大小)`
- 相比`malloc`+手动清零，代码更简洁但**性能稍低**

#### strdup——字符串专用分配

```c
char *strdup(const char *s);

// 示例：复制字符串到堆
char *copy = strdup("Hello, world!");
// 等价于：
// char *copy = malloc(strlen("Hello, world!") + 1);
// strcpy(copy, "Hello, world!");
```

**特点：**
- 自动计算字符串长度（需+1给'\0'）
- 返回堆分配的新字符串副本
- **调用者负责释放**

### 4.4 完整的堆分配示例

```c
char *create_string_good(char ch, int num) {
    // 步骤1: 分配内存（+1 给终止符 '\0'）
    char *new_str = malloc(sizeof(char) * (num + 1));

    // 步骤2: 断言检查
    if (new_str == NULL) {
        return NULL;   // 或使用 assert(new_str != NULL)
    }

    // 初始化字符串
    for (int i = 0; i < num; i++) {
        new_str[i] = ch;
    }
    new_str[num] = '\0';

    return new_str;     // 调用者负责释放
}

int main() {
    char *str = create_string_good('a', 4);
    if (str != NULL) {
        printf("%s\n", str);   // 输出: aaaa

        // 步骤3: 释放内存
        free(str);
    }
    return 0;
}
```

### 4.5 realloc——调整内存大小

```c
void *realloc(void *ptr, size_t new_size);
```

**行为说明：**

1. 尽可能**原地扩展**（如果后续有连续空闲空间）
2. 否则分配新内存、**复制数据**、释放旧内存
3. 返回新内存的地址（可能与原地址不同）

```c
char *str = strdup("Hello");
assert(str != NULL);

// 将 "Hello" 扩展为 "Hello world!"
char *addition = " world!";
size_t new_len = strlen(str) + strlen(addition) + 1;

char *new_str = realloc(str, new_len);
if (new_str == NULL) {
    // 分配失败，原内存仍有效
    free(str);
    return 1;
}
str = new_str;          // 更新指针（重要！）
strcat(str, addition);

printf("%s\n", str);    // 输出: Hello world!
free(str);
```

> **警告**：必须保存`realloc`的返回值！若原地扩展失败，原指针可能失效。

**数组动态扩容示例：**

```c
int *double_array_size(int *arr, int current_len) {
    int new_len = current_len * 2;
    int *new_arr = realloc(arr, new_len * sizeof(int));

    if (new_arr == NULL) {
        // 处理错误，原数组仍然有效
        return arr;
    }
    return new_arr;
}
```

## 五、释放内存与常见陷阱

### 5.1 free函数详解

```c
void free(void *ptr);
```

**重要规则：**

- `ptr`必须是由`malloc`/`calloc`/`realloc`/`strdup`返回的**起始地址**
- 不能释放部分内存（如`ptr + 1`）
- 不能重复释放同一块内存（**double free**）
- 释放后指针变为**悬空指针**，不应再解引用

```c
int *arr = malloc(10 * sizeof(int));
free(arr);
// arr 现在指向已释放的内存（悬空指针）
arr = NULL;  // 好习惯：释放后置空
```

### 5.2 内存泄漏

```c
char *str = strdup("hello");
str = strdup("hi");     // ❌ 内存泄漏！丢失了指向"hello"的指针
```

**泄漏示意图：**

```
初始状态:    str → [ "hello" 的堆内存 ]
重新赋值后:  str → [ "hi" 的堆内存 ]
              [ "hello" 的堆内存 ] ← 无法访问，无法释放 → 泄漏
```

**预防措施：**

- 使用`valgrind --leak-check=full ./program`检测泄漏
- 遵循"谁分配，谁释放"原则
- 在函数文档中明确说明调用者是否需要释放返回值

### 5.3 完整的内存管理示例

```c
// 返回前 len 个 mult 的倍数
int *array_of_multiples(int mult, int len) {
    if (len <= 0) {
        return NULL;
    }

    int *arr = malloc(sizeof(int) * len);
    if (arr == NULL) {
        return NULL;      // 分配失败
    }

    for (int i = 0; i < len; i++) {
        arr[i] = mult * (i + 1);
    }
    return arr;
}

int main() {
    int len = 5;
    int *multiples = array_of_multiples(3, len);

    if (multiples == NULL) {
        fprintf(stderr, "Memory allocation failed\n");
        return 1;
    }

    for (int i = 0; i < len; i++) {
        printf("%d ", multiples[i]);  // 输出: 3 6 9 12 15
    }
    printf("\n");

    free(multiples);      // 释放内存
    return 0;
}
```

## 六、栈与堆的对比

| 特性         | 栈 (Stack)            | 堆 (Heap)                     |
| ------------ | --------------------- | ----------------------------- |
| **管理方式** | 编译器自动管理        | 程序员手动管理（malloc/free） |
| **分配速度** | 快（仅移动栈指针）    | 慢（需要查找空闲块）          |
| **内存限制** | 较小（通常8MB）       | 较大（可达系统内存上限）      |
| **生命周期** | 随函数帧自动创建/销毁 | 从malloc到free                |
| **碎片问题** | 无碎片                | 可能产生内存碎片              |
| **典型用途** | 局部变量、函数参数    | 动态数据结构、跨函数持久数据  |

**选择原则：**

- 默认优先使用**栈**（简单、安全、高效）
- 以下情况考虑**堆**：
  - 数据量过大，可能溢出栈
  - 数据需要在函数调用之间持久存在
  - 运行时才能确定大小
  - 需要动态调整大小

## 七、结构体(Struct)与内存

### 7.1 结构体基础

```c
typedef struct {
    int month;
    int day;
} date;

date today = {1, 28};      // 初始化
today.month = 1;           // 成员访问
today.day = 28;
```

### 7.2 结构体与指针

```c
void advance_day(date *d) {
    d->day++;              // 等价于 (*d).day++
}

int main() {
    date my_date = {1, 28};
    advance_day(&my_date);  // 传递指针避免拷贝
    printf("%d\n", my_date.day);  // 输出: 29
    return 0;
}
```

### 7.3 结构体大小

```c
typedef struct {
    int x;      // 4字节
    char c;     // 1字节（可能有3字节填充）
} my_struct;

printf("%zu\n", sizeof(my_struct));  // 通常输出8（内存对齐）
```

> **内存对齐**：编译器可能插入填充字节以提高访问效率。

### 7.4 结构体数组

```c
my_struct *arr = malloc(5 * sizeof(my_struct));
if (arr != NULL) {
    arr[0] = (my_struct){10, 'A'};   // 类型指定语法
    arr[0].x = 20;                    // 或逐个赋值
    arr[0].c = 'B';
    free(arr);
}
```

## 八、指针深度操作练习

### 8.1 指针类型层级

```c
char ch = 'r';
char *cptr = &ch;      // cptr 指向 ch
char **strptr = &cptr;  // strptr 指向 cptr
```

**解引用层级：**

```c
ch = 'r';
*cptr = 's';           // 修改 ch 为 's'
**strptr = 't';        // 修改 ch 为 't'（两次解引用）
```

### 8.2 数组与指针的等价操作

```c
char *strs[] = {"hello", "world", "cs107"};
size_t total_len = 0;

// 方式1：数组下标
for (int i = 0; i < 3; i++) {
    total_len += strlen(strs[i]);
}

// 方式2：指针算术（等价）
for (int i = 0; i < 3; i++) {
    total_len += strlen(*(strs + i));
}
```

### 8.3 手写strdup实现

```c
#include <stdlib.h>
#include <string.h>
#include <assert.h>

char *my_strdup(const char *str) {
    // 计算长度（+1 给 '\0'）
    size_t len = strlen(str) + 1;

    // 分配内存
    char *heap_str = malloc(len);

    // 断言检查
    if (heap_str == NULL) {
        return NULL;     // 或 assert(heap_str != NULL)
    }

    // 复制字符串
    strcpy(heap_str, str);

    return heap_str;
}
```

### 8.4 strcat_extend 完整实现

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <assert.h>

/**
 * 扩展堆字符串并连接另一个字符串
 * @param heap_str 堆分配的源字符串（将被重新分配）
 * @param concat_str 要连接的字符串
 * @return 新的堆字符串指针（调用者负责释放）
 */
char *strcat_extend(char *heap_str, const char *concat_str) {
    // 计算新字符串长度（+1 给 '\0'）
    int new_length = strlen(heap_str) + strlen(concat_str) + 1;

    // 重新分配内存
    char *new_str = realloc(heap_str, new_length);
    if (new_str == NULL) {
        // 分配失败，原内存仍有效，但无法扩展
        return heap_str;
    }

    // 连接字符串
    strcat(new_str, concat_str);

    return new_str;
}

int main() {
    char *str = strdup("Hello ");
    assert(str != NULL);

    str = strcat_extend(str, "world!");
    if (str != NULL) {
        printf("%s\n", str);  // 输出: Hello world!
        free(str);
    }

    return 0;
}
```

## 九、调试工具：Valgrind

### 9.1 检测内存泄漏

```bash
valgrind --leak-check=full --show-leak-kinds=all ./your_program
```

**输出解读：**

- `definitely lost`：确定泄漏，必须修复
- `indirectly lost`：间接泄漏（通常因丢失指向某结构的指针）
- `possibly lost`：可能泄漏（指针指向块中间）
- `still reachable`：程序结束时仍可访问（通常可忽略）

### 9.2 检测内存错误

- 访问已释放的内存
- 数组越界
- 使用未初始化的内存
- 重复释放

### 9.3 使用 Valgrind 调试示例

```c
// memory_error_demo.c
#include <stdlib.h>

int main() {
    int *arr = malloc(5 * sizeof(int));
    arr[5] = 100;           // 越界访问
    free(arr);
    free(arr);              // 双重释放
    return 0;
}
```

```bash
# 编译并运行 Valgrind
gcc -g memory_error_demo.c -o demo
valgrind --leak-check=full ./demo
```

## 十、最佳实践与常见陷阱总结

### ✅ 推荐做法

```c
// 1. 分配后立即检查
int *ptr = malloc(size);
if (ptr == NULL) {
    // 处理错误
}

// 2. 使用后立即释放并置空
free(ptr);
ptr = NULL;

// 3. 明确文档说明
// 返回值：指向堆分配的字符串，调用者必须调用 free()
```

### ❌ 常见错误

| 错误类型       | 示例                           | 后果           |
| -------------- | ------------------------------ | -------------- |
| 返回栈地址     | `return &local_var;`           | 悬空指针       |
| 忘记释放       | `malloc()` 没有对应的 `free()` | 内存泄漏       |
| 重复释放       | `free(p); free(p);`            | 双重释放错误   |
| 释放非堆内存   | `free(&stack_var);`            | 未定义行为     |
| 释放部分内存   | `free(ptr+1);`                 | 未定义行为     |
| 使用已释放内存 | `free(p); *p = 5;`             | use-after-free |

### 9.4 正确释放内存的模式

```c
// 模式1：循环中分配必须循环中释放
for (int i = 0; i < 5; i++) {
    int *num = malloc(sizeof(int));
    *num = i;
    printf("%d\n", *num);
    free(num);  // 必须在循环内释放
}

// 模式2：多个指针指向同一内存只释放一次
char *str = strdup("Hello");
char *ptr = str;  // ptr 和 str 指向同一内存
// ... 使用 str 或 ptr ...
free(str);        // 只释放一次
// free(ptr);     // ❌ 错误：重复释放

// 模式3：结构体中的动态成员
typedef struct {
    char *name;
    int id;
} Person;

Person *create_person(const char *name, int id) {
    Person *p = malloc(sizeof(Person));
    if (p == NULL) return NULL;

    p->name = strdup(name);
    if (p->name == NULL) {
        free(p);
        return NULL;
    }
    p->id = id;
    return p;
}

void free_person(Person *p) {
    if (p != NULL) {
        free(p->name);  // 先释放内部动态成员
        free(p);        // 再释放结构体本身
    }
}
```

## 十一、综合练习

### 11.1 数组压缩函数

```c
#include <stdio.h>
#include <stdlib.h>
#include <assert.h>

#define SIZE_THRESHOLD 5

/**
 * 压缩数组：
 * - 若长度 <= SIZE_THRESHOLD，两两合并（求和）
 * - 否则，三三合并（求和）
 * @param values 输入数组
 * @param n 输入数组长度
 * @param out_n 输出数组长度（通过指针返回）
 * @return 堆分配的压缩后数组，调用者负责释放
 */
int *compress_in_batches(const int *values, int n, int *out_n) {
    if (n == 0) {
        *out_n = 0;
        return NULL;
    }

    int merge_size = (n <= SIZE_THRESHOLD) ? 2 : 3;

    // 检查长度是否为 merge_size 的倍数
    if (n % merge_size != 0) {
        *out_n = 0;
        return NULL;
    }

    int out_len = n / merge_size;
    int *compressed = malloc(sizeof(int) * out_len);
    if (compressed == NULL) {
        *out_n = 0;
        return NULL;
    }

    for (int i = 0; i < out_len; i++) {
        int sum = 0;
        for (int j = 0; j < merge_size; j++) {
            sum += values[i * merge_size + j];
        }
        compressed[i] = sum;
    }

    *out_n = out_len;
    return compressed;
}

int main() {
    int arr1[] = {1, 2, 3, 4};  // 长度 4 <= 5，两两合并: (1+2)=3, (3+4)=7
    int out_len;
    int *result = compress_in_batches(arr1, 4, &out_len);

    if (result != NULL) {
        for (int i = 0; i < out_len; i++) {
            printf("%d ", result[i]);  // 输出: 3 7
        }
        printf("\n");
        free(result);
    }

    return 0;
}
```

### 11.2 读取任意长度行（作业3核心）

```c
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include <assert.h>

/**
 * 从文件中读取一行（任意长度）
 * @param fp 文件指针
 * @return 堆分配的行字符串（不含换行符），调用者负责释放
 */
char *read_line(FILE *fp) {
    int capacity = 16;
    int size = 0;
    char *buffer = malloc(capacity);
    assert(buffer != NULL);

    int ch;
    while ((ch = fgetc(fp)) != EOF && ch != '\n') {
        if (size + 1 >= capacity) {
            capacity *= 2;
            char *new_buffer = realloc(buffer, capacity);
            if (new_buffer == NULL) {
                free(buffer);
                return NULL;
            }
            buffer = new_buffer;
        }
        buffer[size++] = ch;
    }

    if (size == 0 && ch == EOF) {
        free(buffer);
        return NULL;
    }

    buffer[size] = '\0';
    return buffer;
}
```

## 自测问题


**Q1: 以下代码有什么问题？**

```c
int *get_array() {
    int arr[10] = {0};
    return arr;
}
```

**A1:** 返回指向栈内存的指针。函数返回后，栈帧被销毁，`arr` 不再有效。应使用 `malloc` 在堆上分配。

---

**Q2: `sizeof(arr)` 和 `sizeof(ptr)` 有什么区别？**

**A2:** `sizeof(arr)` 返回整个数组的字节数；`sizeof(ptr)` 在64位系统上始终返回8（指针大小），无论它指向什么。

---

**Q3: `realloc` 失败时会返回什么？原来的内存还在吗？**

**A3:** `realloc` 失败时返回 `NULL`，原来的内存**保持不变**（未释放、未移动）。

---

**Q4: 如何避免内存泄漏？**

**A4:**
1. 每个 `malloc`/`calloc`/`strdup` 都有对应的 `free`
2. 使用 Valgrind 检测
3. 释放后将指针置 `NULL`
4. 遵循清晰的"所有权"规则

---

**Q5: 以下代码的输出是什么？**

```c
int x = 10;
int *p = &x;
int **pp = &p;
**pp = 20;
printf("%d", x);
```

**A5:** 输出 `20`。`**pp = 20` 通过双重解引用修改了 `x` 的值。

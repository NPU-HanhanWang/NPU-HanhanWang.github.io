### 1.2 内存的核心问题

**什么是"内存"？**

在计算机系统中，"内存"是一个广义概念，涵盖了从CPU寄存器到远程网络缓存的所有数据存储设备。大多数计算需要：

- **复杂输入**：程序需要处理大量外部数据
- **副作用（Side Effects）** ：程序执行会产生状态变化，需要持久化存储

这些数据都存储在"内存"中，但不同层次的内存具有截然不同的性能特征。

### 1.3 内存的读写操作

**写操作（Write/Store）** ：
- 将数据从CPU传输到内存
- 汇编示例：`movq %rax, 8(%rsp)` — 将 `%rax` 的值存储到栈指针偏移8字节的位置

**读操作（Read/Load）** ：
- 将数据从内存传输到CPU
- 汇编示例：`movq 8(%rsp), %rax` — 从栈指针偏移8字节的位置读取数据到 `%rax`

<details>
<summary><b>💡 补充：内存操作的底层视角</b></summary>

从硬件角度看，内存读/写操作涉及：

1. **地址解码**：CPU将虚拟地址转换为物理地址（通过MMU）
2. **总线事务**：通过系统总线传输地址和控制信号
3. **数据选通**：内存控制器根据地址选择对应的存储单元
4. **数据传输**：数据通过数据总线在CPU和内存之间传输

读操作通常比写操作更快，因为写操作需要额外的"写回"确认周期。
</details>

### 1.4 CPU-内存鸿沟（The CPU-Memory Gap）

**核心问题**：CPU速度与内存速度之间的差距持续扩大。

```
速度层级（从快到慢）：
┌─────────────────────────────────────────────┐
│  CPU寄存器  ←  最快，但容量最小              │
│  ↓  速度差距约 1-2 个数量级                  │
│  SRAM（L1/L2/L3缓存）                       │
│  ↓  速度差距约 1-2 个数量级                  │
│  DRAM（主存）                               │
│  ↓  速度差距约 2-3 个数量级                  │
│  磁盘（机械硬盘）                           │
│  ↓  速度差距约 3-4 个数量级                  │
│  远程存储/网络                              │
└─────────────────────────────────────────────┘
```

**"内存墙"（Memory Wall）/ 冯·诺依曼瓶颈**：

- 概念：计算速度与数据存取速度之间的性能差距
- 本质：CPU运算速度增长远快于内存访问速度增长

**三种解决策略**：

1. **建立层次结构**（Memory Hierarchy）—— 本课程重点（213）
2. **寻找其他工作**（多线程/异步）—— 涵盖于 346、418
3. **移动计算**（近数据计算）—— 涵盖于 346、7xx

<details>
<summary><b>📊 具体数据对比（2015年参考值）</b></summary>

| 存储类型 | 访问时间    | 相对DRAM               |
| -------- | ----------- | ---------------------- |
| SRAM     | ~0.3-0.5 ns | ~50-100x 更快          |
| DRAM     | ~20-50 ns   | 1x（基准）             |
| 磁盘     | ~3-8 ms     | ~100,000-250,000x 更慢 |

注意：这些数字随着技术进步不断变化，但**相对差距**持续存在。
</details>


## 二、随机存取存储器（RAM）

### 2.1 RAM 的基本特征

RAM 通常封装为芯片，或嵌入处理器芯片中。

- **基本存储单元**：通常为 **1 个 cell = 1 bit**
- **多个 RAM 芯片** 组成完整的存储系统
- 两种主要类型：**SRAM** 和 **DRAM**

### 2.2 SRAM vs DRAM 技术对比

```
┌─────────────────────────────────────────────────────────────────┐
│                    SRAM 结构                                    │
│  ┌─────────────────────────────────────────┐                  │
│  │  6 个晶体管 / bit                       │                  │
│  │  状态保持：无限期（只要供电）           │                  │
│  │  无需刷新                               │                  │
│  │  访问时间：~1x（基准，最快）            │                  │
│  │  成本：~100x（相对DRAM）               │                  │
│  │  主要用途：CPU缓存（L1/L2/L3）         │                  │
│  └─────────────────────────────────────────┘                  │
├─────────────────────────────────────────────────────────────────┤
│                    DRAM 结构                                    │
│  ┌─────────────────────────────────────────┐                  │
│  │  1 个晶体管 + 1 个电容 / bit            │                  │
│  │  电容垂直排列（3D结构）                 │                  │
│  │  必须周期性刷新（~每64ms）              │                  │
│  │  访问时间：~10x（相对SRAM）             │                  │
│  │  成本：~1x（基准，最便宜）              │                  │
│  │  主要用途：主存（Main Memory）          │                  │
│  └─────────────────────────────────────────┘                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 SRAM vs DRAM 详细对比表

| 特性                      | SRAM          | DRAM                |
| ------------------------- | ------------- | ------------------- |
| **每bit晶体管数**         | 6 或 8        | 1                   |
| **访问时间**              | 1×（快）      | 10×（较慢）         |
| **需要刷新？**            | 否            | 是（每~64ms）       |
| **EDC（错误检测与纠正）** | 可选          | 通常需要            |
| **成本**                  | ~100×         | 1×                  |
| **典型应用**              | 缓存（Cache） | 主存（Main Memory） |

### 2.4 技术趋势与挑战

**SRAM 趋势**：
- 随半导体工艺缩放（Scaling）
- 正接近物理极限（漏电流、稳定性问题）

**DRAM 趋势**：
- 缩放受限于保持最小电容的需求
- 高宽比（Aspect Ratio）限制电容可以做到多深
- 同样接近物理极限

**新兴技术**：3D XPoint（Intel Optane）、MRAM、ReRAM 等非易失性内存（NVM）正在兴起。

<details>
<summary><b>🔬 补充：DRAM 刷新机制详解</b></summary>

DRAM 的电容器会随时间泄漏电荷。为防止数据丢失：

1. **刷新周期**：通常每 64ms 刷新所有行
2. **刷新操作**：读取每一行数据，然后重新写入（恢复电荷）
3. **刷新开销**：约占总带宽的 1-2%
4. **影响**：刷新期间无法正常访问内存（"刷新周期"）

现代 DRAM 采用"分布式刷新"（Distributed Refresh），将刷新操作分散到时间中，减少对性能的影响。
</details>


## 三、局部性原理（Locality of Reference）

### 3.1 核心概念

**局部性原理**：程序倾向于使用**近期使用过的**数据/指令，或**地址邻近**的数据/指令。

| 类型                                | 定义                                     | 示例                                         |
| ----------------------------------- | ---------------------------------------- | -------------------------------------------- |
| **时间局部性**（Temporal Locality） | 近期引用的项很可能在不久的将来再次被引用 | 循环中的计数器变量 `i`、`sum`                |
| **空间局部性**（Spatial Locality）  | 地址邻近的项倾向于在相近时间被引用       | 顺序访问数组元素 `a[0]`, `a[1]`, `a[2]`, ... |

```
时间局部性示例：
┌─────────────────────────────────────────────────┐
│  for (int i = 0; i < n; i++) {                 │
│      sum += a[i];   ← 每次迭代都访问 sum       │
│  }                    ← 时间局部性！           │
└─────────────────────────────────────────────────┘

空间局部性示例：
┌─────────────────────────────────────────────────┐
│  for (int i = 0; i < n; i++) {                 │
│      sum += a[i];   ← 顺序访问 a[0], a[1], ... │
│  }                    ← 空间局部性！           │
└─────────────────────────────────────────────────┘
```

### 3.2 局部性分析示例

```c
long sum = 0;
for (int i = 0; i < n; i++) {
    sum += a[i];
}
return sum;
```

**数据引用分析**：
- `a[i]`：顺序访问 → **空间局部性**（步长为1的引用模式）
- `sum`：每次迭代都访问 → **时间局部性**

**指令引用分析**：
- 顺序执行指令 → **空间局部性**
- 循环反复执行 → **时间局部性**

### 3.3 关键技能：代码局部性评估

> 专业程序员的核心技能：能够查看代码并定性评估其局部性。

**问题1**：以下函数对数组 `a` 是否具有良好的局部性？（假设数组按行主序布局）

```c
// 二维数组访问：按行遍历
for (int i = 0; i < M; i++) {
    for (int j = 0; j < N; j++) {
        sum += a[i][j];
    }
}
```

**答案**：✅ **是，具有良好的局部性**

- 内层循环 `j` 在行内连续移动 → **步长为1** → 极好的空间局部性
- 每个元素只访问一次 → 没有时间局部性（对于数组元素）
- 变量 `sum` 有良好的时间局部性

内存布局（行主序）：
```
行0: [a[0][0]] [a[0][1]] [a[0][2]] ... [a[0][N-1]]
行1: [a[1][0]] [a[1][1]] [a[1][2]] ... [a[1][N-1]]
      ↑  ← 访问顺序（从左到右）
```

**问题2**：以下函数对数组 `a` 的局部性如何？

```c
// 二维数组访问：按列遍历
for (int j = 0; j < N; j++) {
    for (int i = 0; i < M; i++) {
        sum += a[i][j];
    }
}
```

**答案**：❌ **局部性较差（步长为 N）**

- 每次访问跳 `N` 个元素 → **步长为 N** → 较差的空间局部性
- 如果 `N` 很大，每次访问都在不同的缓存块 → 缓存抖动

> ⚠️ **注意**：如果 `N` 非常小（例如 `N = 2`），则步长很小，局部性会变好。但在实际大规模数据中，步长 N 通常远大于缓存块大小。

**问题3**：如何调整循环顺序，使3维数组具有步长为1的引用模式？

```c
// 原始代码（局部性差）
for (int i = 0; i < M; i++) {
    for (int j = 0; j < N; j++) {
        for (int k = 0; k < P; k++) {
            sum += a[i][j][k];
        }
    }
}
```

**答案**：将 `k` 作为**最内层循环**（即 `i` 和 `j` 在外层）

> 关键在于：数组在内存中是**行主序**存储，最内层循环应该沿着**最后一个维度**遍历，这样才能保证内存地址连续。

```c
// 优化后（良好的空间局部性）
for (int i = 0; i < M; i++) {
    for (int j = 0; j < N; j++) {   // j 在外层
        for (int k = 0; k < P; k++) { // k 在最内层 → 步长为1
            sum += a[i][j][k];
        }
    }
}
```


## 四、内存层次结构（Memory Hierarchy）

### 4.1 三个核心观察

1. **快速存储技术**：每字节成本更高、容量更小、功耗更大（产生更多热量）
2. **CPU-主存速度差距**：持续扩大
3. **程序局部性**：良好编写的程序表现出优秀的局部性

**关键洞察**：这三个特性在大多数程序中互补，为构建内存层次结构提供了理论基础。

### 4.2 层次结构的工作原理

```
      级别0:  CPU寄存器                 ← 最快，最小，最贵
            ↕  (缓存关系)
      级别1:  L1 缓存（SRAM）
            ↕  (缓存关系)
      级别2:  L2 缓存（SRAM）
            ↕  (缓存关系)
      级别3:  L3 缓存（SRAM）
            ↕  (缓存关系)
      级别4:  主存（DRAM）
            ↕  (缓存关系)
      级别5:  本地磁盘（SSD/HDD）
            ↕  (缓存关系)
      级别6:  远程存储/网络
                                        ← 最慢，最大，最便宜
```

**基本思想**：对于每一对相邻级别 `(k, k+1)`：
- 级别 k：**更小、更快、更贵**的设备
- 级别 k+1：**更大、更慢、更便宜**的设备
- 级别 k **缓存（Cache）** 级别 k+1 的部分数据

### 4.3 为什么层次结构有效？

- **局部性原理**：程序访问级别 k 的数据比访问级别 k+1 的数据更频繁
- 因此，级别 k+1 的存储设备可以**更慢**，从而**更大、更便宜**

**大思想（理想化）** ：
> 内存层次结构创造了一个巨大的存储池，成本与底层廉价存储相当，但能够以顶层快速存储的速度为程序提供数据。

### 4.4 缓存（Cache）的定义

> **缓存**：一个较小、较快的存储设备，作为较大、较慢设备中部分数据的"暂存区"。

**缓存与内存的区别**：

| 特性             | 缓存（Cache）                   | 内存（Memory）               |
| ---------------- | ------------------------------- | ---------------------------- |
| **对软件可见性** | 不可见（透明）                  | 可见                         |
| **管理方式**     | 硬件自动管理（load/store 触发） | 软件可直接寻址               |
| **优化方式**     | 性能提升无需软件改动            | 需要软件优化（如局部性优化） |

> ⚠️ **注意**：现代CPU有一些指令可以管理缓存（如预取 prefetch、失效 invalidate、分区 partition），但大多数情况下缓存对软件是透明的。


## 五、通用缓存概念

### 5.1 缓存的工作模型

```
        CPU 请求数据
              │
              ▼
    ┌─────────────────────┐
    │  级别 k 缓存         │  ← 小、快、贵
    │  （存储部分块）      │
    └─────────┬───────────┘
              │
              ▼ 如果数据不在缓存中（Miss）
    ┌─────────────────────┐
    │  级别 k+1 内存       │  ← 大、慢、便宜
    │  （分区为"块"）      │
    └─────────────────────┘
```

**关键术语**：
- **块（Block）** ：级别 k+1 内存被划分为固定大小的块（cache block）
- **缓存行（Cache Line）** ：缓存中存储块的位置

### 5.2 缓存命中（Hit）

```
CPU 需要数据块 b
    │
    ▼
检查缓存 ──► 块 b 在缓存中 ──► ✅ 命中！
    │
    └──► 直接从缓存返回数据
```

### 5.3 缓存未命中（Miss）

```
CPU 需要数据块 b
    │
    ▼
检查缓存 ──► 块 b 不在缓存中 ──► ❌ 未命中！
    │
    ├──► 从级别 k+1 内存取回块 b
    ├──► 将块 b 存入缓存
    │        │
    │        ├── 放置策略（Placement Policy）：决定 b 放在缓存的哪个位置
    │        └── 替换策略（Replacement Policy）：决定驱逐哪个块（受害者）
    │
    └──► 将数据返回 CPU
```

### 5.4 三种缓存未命中类型

| 类型                                   | 描述                              | 示例                                     |
| -------------------------------------- | --------------------------------- | ---------------------------------------- |
| **冷启动缺失**（Cold/Compulsory Miss） | 第一次访问该块                    | 程序刚开始执行，缓存为空                 |
| **容量缺失**（Capacity Miss）          | 工作集（Working Set）大于缓存大小 | 正在处理的数据集超过缓存容量             |
| **冲突缺失**（Conflict Miss）          | 多个数据对象映射到同一个缓存位置  | 访问块 0, 8, 0, 8, ...（映射到同一位置） |

**冲突缺失的详细说明**：

大多数缓存将级别 k+1 的块限制为级别 k 的块位置的**小子集**（有时是单例）。

- 例如：级别 k+1 的块 `i` 必须放置在级别 k 的块 `(i mod 4)` 位置
- 当引用块 `0, 8, 0, 8, 0, 8` 时，每次都映射到位置 0 → 每次都缺失

**三种缺失的层级关系**：
```
冷启动缺失（Cold Miss）
    │
    ├── 使用无限大缓存且无放置限制 → 仅冷启动缺失
    │
    ├── 加上有限大小缓存 → 额外产生容量缺失
    │
    └── 加上实际放置策略限制 → 额外产生冲突缺失
```

### 5.5 内存层次结构中的缓存实例

| 缓存类型           | 缓存内容  | 存储位置       | 延迟（周期）   | 管理方        |
| ------------------ | --------- | -------------- | -------------- | ------------- |
| **寄存器**         | 4-8字节字 | CPU核心        | 0              | 编译器        |
| **TLB**            | 地址翻译  | 片上TLB        | 0              | 硬件MMU       |
| **L1缓存**         | 64字节块  | 片上L1         | ~4             | 硬件          |
| **L2缓存**         | 64字节块  | 片上L2         | ~10            | 硬件          |
| **虚拟内存**       | 4KB页     | 主存           | ~100           | 硬件+OS       |
| **Buffer缓存**     | 文件部分  | 主存           | ~100           | OS            |
| **磁盘缓存**       | 磁盘扇区  | 磁盘控制器     | ~100,000       | 磁盘固件      |
| **网络Buffer缓存** | 文件部分  | 本地磁盘       | ~10,000,000    | NFS客户端     |
| **浏览器缓存**     | Web页面   | 本地磁盘       | ~10,000,000    | 浏览器        |
| **Web缓存**        | Web页面   | 远程服务器磁盘 | ~1,000,000,000 | Web代理服务器 |

### 5.6 工作集（Working Set）与缓存

> **工作集**：程序当前"正在工作"的数据集合

**"当前"的定义**：取决于上下文（例如，当前循环、当前函数、当前进程）

工作集包括：
- **数据访问**：变量、数组、结构体
- **指令访问**：程序代码

**缓存的利用**：
- **时间局部性** → 缓存最近使用的数据
- **空间局部性** → 以块为单位复制数据

> **关键结论**：局部性减少工作集大小 → 工作集适合缓存时，缓存效率最高

### 5.7 冷启动缺失数量分析

**示例代码**：
```c
long sum = 0;
for (int i = 0; i < 16; i++) {
    sum += a[i];
}
return sum;
```

**影响因素**：
1. 是否有元素已在缓存中？
2. 每个缓存块能容纳多少个元素？
3. 数组起始地址是否与缓存块起始地址对齐？

**示例场景**：
- 缓存初始为空（所有元素都在主存中）
- 每个缓存块可容纳 8 个元素（64字节块 / 8字节long）
- 数组与缓存块起始地址对齐

**结果**：**2 次冷启动缺失**

```
内存布局（缓存块对齐）：
┌──────────────┬──────────────┬──────────────┬──────────────┐
│  块0          │  块1          │  块2          │  块3          │
├──────────────┼──────────────┼──────────────┼──────────────┤
│ a[0] a[1] ...│ a[8] a[9] ...│ ...          │ ...          │
│ a[7]          │ a[15]        │              │              │
└──────────────┴──────────────┴──────────────┴──────────────┘
    ↑ 缺失1         ↑ 缺失2
  加载块0        加载块1
  (a[0]-a[7])    (a[8]-a[15])
  全部命中        全部命中
```


## 六、存储技术趋势

### 6.1 存储技术概览

**磁存储**（磁盘）：
- 在磁介质上存储数据
- 电机-机械访问方式
- 容量大、成本低、速度慢

**非易失性存储器**（Flash）：
- 存储为持久电荷
- 3D结构实现
- 100+ 层单元堆叠
- 每单元可存储 3-4 位数据

### 6.2 磁盘驱动器

**磁盘几何结构**：
- **盘片（Platter）** ：每个磁盘包含一个或多个盘片
- **表面（Surface）** ：每个盘片有两个表面
- **磁道（Track）** ：每个表面由同心环（磁道）组成
- **扇区（Sector）** ：每个磁道由扇区组成，扇区之间由间隙分隔

```
俯视图（单盘片）：
┌─────────────────────────────────┐
│    ┌───────────────────────┐    │
│    │    ┌──────────────┐   │    │
│    │    │   ┌──────┐    │   │    │
│    │    │   │ 扇区 │    │   │    │
│    │    │   └──────┘    │   │    │
│    │    │  磁道          │   │    │
│    │    └──────────────┘   │    │
│    │   主轴                │    │
│    └───────────────────────┘    │
│        读/写磁头                 │
└─────────────────────────────────┘
```

**磁盘访问时间分解**：

```
总访问时间 = 寻道时间 + 旋转延迟 + 传输时间

┌─────────────┬──────────────────────────────────────────┐
│ 寻道时间    │ 磁头移动到目标磁道所需时间               │
│ (Seek Time) │ 通常 3-9 ms                            │
├─────────────┼──────────────────────────────────────────┤
│ 旋转延迟    │ 等待目标扇区旋转到磁头下方               │
│ (Rotation)  │ 平均 = 0.5 × 旋转周期                   │
│             │ 7200 RPM → 平均 4 ms                   │
├─────────────┼──────────────────────────────────────────┤
│ 传输时间    │ 实际读取扇区数据的时间                   │
│ (Transfer)  │ 相对较小（~0.02 ms）                   │
└─────────────┴──────────────────────────────────────────┘
```

> ⚠️ **注意**：磁盘访问时间由**寻道时间**和**旋转延迟**主导。第一个bit的访问最昂贵，后续bits几乎"免费"。

**磁盘访问时间示例**：
- 7200 RPM，平均寻道 9ms，每磁道 400 个扇区
- 平均旋转延迟 = 0.5 × (60/7200) × 1000 = **4 ms**
- 平均传输时间 = (60/7200) × (1/400) × 1000 = **0.02 ms**
- 总访问时间 = 9 + 4 + 0.02 = **13.02 ms**

**对比速度**：
- SRAM 访问时间：~4 ns / doubleword
- DRAM 访问时间：~60 ns
- 磁盘访问时间：~13 ms
- **磁盘比 SRAM 慢约 3,250,000 倍！**
- **磁盘比 DRAM 慢约 216,000 倍！**

### 6.3 非易失性存储器（NVM）

**ROM（只读存储器）**：
- 生产时编程，不可更改
- 存储固件（BIOS、磁盘控制器、网卡、显卡等）

**EEPROM（电可擦除PROM）**：
- 支持电擦除和重新编程
- Flash：EEPROM 的一种，支持块级擦除
- 约 100,000 次擦写后磨损

**3D XPoint（Intel Optane）**：
- 新兴非易失性内存技术
- 新材料和存储机制
- 介于 DRAM 和 Flash 之间的性能和成本

**非易失性存储用途**：
1. 固件程序（BIOS、控制器固件）
2. 固态硬盘（SSD，取代旋转磁盘）
3. 磁盘缓存

### 6.4 固态硬盘（SSD）

```
SSD 内部结构：
┌─────────────────────────────────────────────┐
│  ┌─────────────────────────────────────────┐ │
│  │  页（Page）：4KB-16KB（读写单元）       │ │
│  ├─────────────────────────────────────────┤ │
│  │  块（Block）：32-128 页（擦除单元）     │ │
│  └─────────────────────────────────────────┘ │
│    读/写以页为单位                          │
│    擦除以块为单位                          │
│    约 10,000 次写入后块磨损                 │
└─────────────────────────────────────────────┘
```

**Flash 工作原理**：
- 使用"浮栅晶体管"（Floating Gate Transistor）存储电荷
- 极高的存储密度：每晶体管多位
- 读/写以块为单位
- 写入较慢：10-100 μs
- 写入次数有限：编程/擦除需要高电压，会损坏晶体管

**SSD 性能基准**（Samsung 970 EVO Plus）：

| 操作类型           | 吞吐量     |
| ------------------ | ---------- |
| 顺序读             | 2,221 MB/s |
| 顺序写             | 1,912 MB/s |
| 随机读（单队列）   | 61.7 MB/s  |
| 随机写（单队列）   | 165 MB/s   |
| 随机读（深度队列） | 947 MB/s   |
| 随机写（深度队列） | 1,028 MB/s |

> **观察**：
> - 顺序访问 > 随机访问（内存层次结构的共同主题）
> - 深度队列（Deep Queue）提高吞吐量（隐藏延迟）
> - 随机写入较复杂（需要块擦除和写缓存）

**SSD vs 传统磁盘**：

| 特性     | SSD                                 | 传统磁盘         |
| -------- | ----------------------------------- | ---------------- |
| **优势** | 无移动部件 → 更快、功耗更低、更坚固 | 更便宜（每字节） |
| **劣势** | 磨损（wear out）                    | 较慢、功耗更高   |
| **缓解** | 磨损均衡逻辑（Wear Leveling）       | —                |
| **寿命** | ~600 次全盘写入（典型）             | 理论无限         |

**旋转磁盘仍然使用场景**：
- 海量存储（视频、大数据集/数据库）
- 廉价存储（台式机）

## 七、补充：DRAM 与内存模块的底层细节

### 7.1 d×w DRAM 结构

> **d×w DRAM**：总共有 `d·w` 位，组织为 `d` 个超单元（supercell），每个超单元大小为 `w` 位。

```
d×w DRAM 组织结构：
┌──────────────────────────────────────────────────────┐
│  行0  │ 超单元(0,0) │ 超单元(0,1) │ ... │ 超单元(0, w-1) │
│  行1  │ 超单元(1,0) │ 超单元(1,1) │ ... │ 超单元(1, w-1) │
│  ...  │    ...      │    ...      │ ... │    ...        │
│  行d-1│ 超单元(d-1,0)│ ...        │ ... │ 超单元(d-1, w-1)│
└──────────────────────────────────────────────────────┘
   ↑                                                   ↑
   行地址 (RAS)                                      列地址 (CAS)
```

### 7.2 DRAM 读取操作

读取超单元 `(2, 1)` 的过程：

**步骤1**：行访问选通（RAS = Row Access Strobe）
- 选择行 2
- 将行 2 全部复制到**行缓冲区**（Row Buffer）

**步骤2**：列访问选通（CAS = Column Access Strobe）
- 选择列 1
- 超单元 `(2, 1)` 从缓冲区复制到数据线，最终返回 CPU

**步骤3**：写回刷新
- 所有数据写回到行中（提供刷新功能）

```
RAS 操作（步骤1）：
┌─────────┐
│ DRAM阵列 │─── 选择行 2 ───→ 复制整行到行缓冲区
└─────────┘

CAS 操作（步骤2）：
┌─────────┐
│ 行缓冲区 │─── 选择列 1 ───→ 输出超单元(2,1)到数据线
└─────────┘
```

### 7.3 内存模块

多个 DRAM 芯片组合成内存模块（如 DIMM）：

```
内存模块（例如：8 个 DRAM 芯片 × 8 位 = 64位数据总线）：
┌──────────────────────────────────────────────────────────────┐
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ │
│  │ DRAM │ │ DRAM │ │ DRAM │ │ DRAM │ │ DRAM │ │ DRAM │ │ DRAM │ │ DRAM │ │
│  │ 芯片1│ │ 芯片2│ │ 芯片3│ │ 芯片4│ │ 芯片5│ │ 芯片6│ │ 芯片7│ │ 芯片8│ │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ └──────┘ │
│      │        │        │        │        │        │        │        │    │
│      └────────┴────────┴────────┴────────┴────────┴────────┴────────┘    │
│                             64位数据总线                                   │
└──────────────────────────────────────────────────────────────┘
```

### 7.4 增强型 DRAM

自 1970 年 Intel 商业化 DRAM 以来，DRAM 单元的操作方式基本未变。变化的是**接口逻辑**和**I/O 速度**：

| 类型                        | 特点                          |
| --------------------------- | ----------------------------- |
| **SDRAM**（同步DRAM）       | 使用时钟信号代替异步控制      |
| **DDR SDRAM**（双倍数据率） | 双沿时钟，每引脚每周期传输2位 |
| **DDR2**                    | 预取缓冲区大小：4位           |
| **DDR3**                    | 预取缓冲区大小：8位           |
| **DDR4**                    | 预取缓冲区大小：16位          |

> 截至 2015 年，Intel Core i7 支持 DDR3 和 DDR4 SDRAM。


## 八、补充：I/O 总线与磁盘操作

### 8.1 I/O 总线架构

```
CPU芯片
┌─────────────────────────────────────────────────────────┐
│  ┌─────────┐    ┌─────────┐                           │
│  │  CPU核心 │    │  I/O桥  │───┬──→ 内存总线 ──→ 主存 │
│  └─────────┘    └─────────┘   │                       │
│       │              │         │                       │
│       └──────────────┘         │                       │
│                                │                       │
│                          I/O总线（共享）               │
│                                │                       │
│                          ┌─────┴─────┐                 │
│                          ▼           ▼                 │
│                      ┌────────┐ ┌────────┐             │
│                      │ 磁盘   │ │ 网卡   │             │
│                      │ 控制器 │ │        │             │
│                      └────────┘ └────────┘             │
└─────────────────────────────────────────────────────────┘
```

> **总线（Bus）** ：一组并行导线，传输地址、数据和控制信号。总线通常由多个设备共享。

### 8.2 磁盘扇区读取过程

**步骤1**：
- CPU 通过 I/O 总线向磁盘控制器发送命令
- 命令包括：读取操作、逻辑块地址（LBA）

**步骤2**：
- 磁盘控制器将 LBA 转换为物理地址（柱面、磁头、扇区）
- 读取数据到磁盘控制器内部缓冲区
- 控制器计算数据校验和（ECC 验证）

**步骤3**：
- 数据通过 I/O 总线从磁盘控制器 DMA（直接内存访问）传输到主存
- DMA 传输不占用 CPU 周期


## 九、总结

### 核心要点回顾

1. **CPU-内存速度差距**：持续扩大，是计算机系统设计的核心挑战
2. **局部性原理**：时间局部性 + 空间局部性，是层次结构有效的基础
3. **内存层次结构**：利用缓存机制，创造"又快又便宜"的内存幻觉
4. **缓存类型**：冷启动缺失、容量缺失、冲突缺失
5. **SRAM vs DRAM**：速度、成本、刷新机制的权衡
6. **磁盘**：寻道时间 + 旋转延迟主导访问时间
7. **SSD**：无移动部件、更快但会磨损
8. **趋势**：Flash 和 SSD 发展速度超过所有其他存储技术

### 关键公式

**磁盘访问时间**：
```
T_access = T_avg_seek + T_avg_rotation + T_avg_transfer
```

**平均旋转延迟**：
```
T_avg_rotation = 0.5 × (60 / RPM) × 1000 ms
```

**平均传输时间**：
```
T_avg_transfer = (60 / RPM) × (1 / sectors_per_track) × 1000 ms
```

### 最终启示

> 内存层次结构利用**局部性**，通过**缓存**将廉价、大容量、慢速存储与快速、小容量、昂贵存储相结合，创造了**大容量且快速**的存储抽象。

**Flash 存储器**正以惊人的速度发展：
- 3D 堆叠技术使每芯片容量大幅增加
- 性能和寿命持续改善
- 正在逐步替代机械硬盘，但机械硬盘在大容量、低成本场景中仍有优势

## 第一部分：x86-64 汇编基础

### 1.1 AT&T 语法 vs Intel 语法

本课程使用 **AT&T 语法**，与 Intel 语法的主要区别：

| 特征       | AT&T 语法        | Intel 语法      |
| ---------- | ---------------- | --------------- |
| 方向       | `movq Src, Dest` | `mov Dest, Src` |
| 寄存器前缀 | `%rax`           | `rax`           |
| 立即数前缀 | `$0x15213`       | `0x15213`       |
| 内存操作   | `(%rbx)`         | `[rbx]`         |

> **关键记忆**：AT&T 中，源操作数在左，目标操作数在右（与 Intel 相反）。

### 1.2 操作数类型

#### 立即数（Immediate）
- 以 `$` 开头
- 十进制：`$15213`
- 十六进制：`$0x3b6d`

#### 寄存器（Registers）
- 以 `%` 开头
- `%rax`：返回值寄存器
- `%eax`：`%rax` 的低 32 位

#### 内存地址（Memory Locations）
```assembly
(%rbx)              # 简单寻址
0x1c(%rax)          # 位移寻址
0x4(%rcx, %rdi, 4)  # 变址寻址：Mem[Reg[Rc] + 4*Reg[Ri] + 4]
```

### 1.3 寻址模式详解

**位移寻址**：`D(Rb)` → 访问 `Mem[Reg[Rb] + D]`

**变址寻址**：`D(Rb, Ri, S)` → 访问 `Mem[Reg[Rb] + S*Reg[Ri] + D]`

其中 S（比例因子）可以是 1、2、4 或 8。

### 1.4 常用指令示例

| 指令                        | 效果                | 说明                       |
| --------------------------- | ------------------- | -------------------------- |
| `mov %rbx, %rdx`            | `rdx = rbx`         | 寄存器间移动               |
| `add (%rdx), %r8`           | `r8 += *rdx`        | 将 rdx 指向的值加到 r8     |
| `imul $3, %r8`              | `r8 *= 3`           | 立即数乘法                 |
| `sub $1, %r8`               | `r8--`              | 减 1                       |
| `lea (%rdx, %rbx, 2), %rdx` | `rdx = rdx + rbx*2` | **不访问内存**，仅计算地址 |

> **重点**：`lea`（Load Effective Address）**不解引用**，只计算有效地址，这是其与 `mov` 的核心区别。

## 第二部分：控制流与条件跳转

### 2.1 比较指令

```assembly
cmpq Src2, Src1    # 计算 Src1 - Src2，设置条件码但不存储结果
testq Src2, Src1   # 计算 Src1 & Src2，设置条件码
```

### 2.2 条件码

| 标志位 | 全称          | 含义                    |
| ------ | ------------- | ----------------------- |
| CF     | Carry Flag    | 进位/借位（无符号溢出） |
| ZF     | Zero Flag     | 结果为零                |
| SF     | Sign Flag     | 结果为负                |
| OF     | Overflow Flag | 有符号溢出              |

### 2.3 跳转指令条件表

| 指令          | 条件           | 含义                         |
| ------------- | -------------- | ---------------------------- |
| `jmp`         | 无条件         | 无条件跳转                   |
| `je` / `jz`   | ZF             | 相等 / 为零                  |
| `jne` / `jnz` | ~ZF            | 不相等 / 非零                |
| `js`          | SF             | 负数                         |
| `jns`         | ~SF            | 非负数                       |
| `jg`          | ~(SF^OF) & ~ZF | **有符号**大于（大于时跳转） |
| `jge`         | ~(SF^OF)       | **有符号**大于等于           |
| `jl`          | SF^OF          | **有符号**小于               |
| `jle`         | (SF^OF) \| ZF  | **有符号**小于等于           |
| `ja`          | ~CF & ~ZF      | **无符号**大于（高于）       |
| `jb`          | CF             | **无符号**小于（低于）       |

### 2.4 关键理解：`cmp` 的方向

```assembly
cmp %r9, %r10    # 计算 %r10 - %r9
jg 地址          # 如果 %r10 > %r9，则跳转
```

> **记忆口诀**：`cmp A, B` 比较的是 A 和 B，但实际上计算 `B - A`，所以条件跳转针对的是 **第二个操作数**。

### 2.5 示例解析

```assembly
cmp $0x15213, %r12
jge deadbeef
```
**含义**：如果 `%r12 >= 0x15213`，跳转到 `deadbeef`。

```assembly
cmp %rax, %rdi
jae 15213b
```
**含义**：如果无符号数 `%rdi >= %rax`，跳转到 `0x15213b`。

```assembly
test %r8, %r8
jnz *%rsi
```
**含义**：如果 `%r8 != 0`，跳转到 `%rsi` 指向的地址。

## 第三部分：函数调用约定（Calling Conventions）

### 3.1 控制传递

**`call` 指令**：
1. 将返回地址压入栈（`%rsp -= 8`，`Mem[%rsp] = 返回地址`）
2. 跳转到目标标签

**`ret` 指令**：
1. 从栈弹出返回地址（`%rip = Mem[%rsp]`，`%rsp += 8`）
2. 跳转到返回地址

```
┌─────────────────┐  %rsp 指向栈顶
│  返回地址        │
├─────────────────┤
│  被保存的寄存器   │
├─────────────────┤
│  局部变量        │
└─────────────────┘  栈向下生长（地址递减）
```

### 3.2 参数传递

**前 6 个参数**按顺序通过寄存器传递：

| 参数位置 | 寄存器 |
| -------- | ------ |
| 第 1 个  | `%rdi` |
| 第 2 个  | `%rsi` |
| 第 3 个  | `%rdx` |
| 第 4 个  | `%rcx` |
| 第 5 个  | `%r8`  |
| 第 6 个  | `%r9`  |

**剩余参数**压入调用者的栈帧（从右到左压栈）。

### 3.3 返回值

**返回值**统一放在 `%rax` 寄存器中。

### 3.4 调用者保存 vs 被调用者保存

| 类型                                              | 寄存器                                                               | 规则                                      |
| ------------------------------------------------- | -------------------------------------------------------------------- | ----------------------------------------- |
| **调用者保存**（Caller-Saved / Call-Clobbered）   | `%rax`，`%rdi`，`%rsi`，`%rdx`，`%rcx`，`%r8`，`%r9`，`%r10`，`%r11` | 调用者负责在 `call` 前保存，`call` 后恢复 |
| **被调用者保存**（Callee-Saved / Call-Preserved） | `%rbx`，`%rbp`，`%r12`-`%r15`，`%rsp`                                | 被调用者负责使用前保存，返回前恢复        |

**完整寄存器表**：

```
%rax    返回值（调用者保存）
%rbx    被调用者保存
%rcx    第4个参数（调用者保存）
%rdx    第3个参数（调用者保存）
%rsi    第2个参数（调用者保存）
%rdi    第1个参数（调用者保存）
%rbp    被调用者保存
%rsp    栈指针（被调用者保存）
%r8     第5个参数（调用者保存）
%r9     第6个参数（调用者保存）
%r10    临时寄存器（调用者保存）
%r11    临时寄存器（调用者保存）
%r12-%r15  被调用者保存
```

### 3.5 函数调用示例

```assembly
0000000000400540 <multstore>:
  400540: push %rbx           # 保存 %rbx（被调用者保存）
  400541: mov %rdx, %rbx      # 保存目标地址
  400544: call 400550 <mult2> # 调用 mult2(x, y)
  400549: mov %rax, (%rbx)    # 将返回值存储到目标地址
  40054c: pop %rbx            # 恢复 %rbx
  40054d: ret                 # 返回

0000000000400550 <mult2>:
  400550: mov %rdi, %rax      # a 放入 %rax
  400553: imul %rsi, %rax     # a * b
  400557: ret                 # 返回值在 %rax 中返回
```

## 第六部分：Switch 语句与跳转表

### 6.1 跳转表原理

编译器根据以下启发式决定如何翻译 switch：
- Case 数量
- Case 值的稀疏程度

**核心思想**：将输入映射到零基索引，然后用索引在跳转表中查找目标地址，实现 $O(1)$ 的分支跳转。

### 6.2 跳转表汇编实现

**Case 1 - 使用绝对地址**：
```
401151: cmp $0x8, %eax          # 比较 eax 与 8
401154: ja 4011b8               # 如果 > 8，跳转到默认
401156: mov %eax, %eax          # 零扩展
401158: lea 0x0(,%rax,4), %rdx  # rdx = rax * 4（索引*4）
40115f: mov 0x402008(,%rdx,4), %eax  # 从跳转表读取目标地址
```

**Case 2 - 使用偏移量**（更常见）：
```
401151: cmp $0x8, %eax
401154: ja 4011b8               # 默认情况
401156: mov %eax, %eax
401158: lea 0x0(,%rax,4), %rdx  # rdx = 索引 * 4
40115f: lea 0xea1(%rip), %rax   # rax = 跳转表基址
401167: mov (%rdx, %rax, 1), %eax # eax = 跳转表[索引]
40116a: cltq                    # 符号扩展
40116c: lea 0xe95(%rip), %rdx   # rdx = 跳转表基址
401173: add %rdx, %rax          # 最终地址 = 基址 + 偏移量
401176: jmp *%rax               # 间接跳转
```

### 6.3 跳转表结构示意图

```
跳转表（4字节偏移量）：
┌─────────────┐
│ 索引 0: 偏移量A │ ──→ 跳转到 case 0
├─────────────┤
│ 索引 1: 偏移量B │ ──→ 跳转到 case 1
├─────────────┤
│ 索引 2: 偏移量C │ ──→ 跳转到 default
├─────────────┤
│    ...      │
└─────────────┘
    基址 + 偏移量 = 实际跳转地址
```

## 第七部分：拆弹策略与技巧

### 7.1 系统化拆弹流程

1. **观察输入格式**
   - 在 `sscanf` 调用处设置断点
   - 检查 `%rsi` 中的格式字符串

2. **分析汇编代码**
   - 使用 `disassemble` 查看阶段函数
   - 识别关键比较和跳转指令
   - 找出预期的输入条件

3. **推导演算逻辑**
   - 写出伪代码
   - 识别循环、条件、数组操作
   - 理解数据转换过程

4. **构造输入**
   - 从约束条件反推有效输入
   - 使用 GDB 动态验证

### 7.2 伪代码转换技巧

```assembly
# 汇编片段
400540: mov 0x8(%rsp), %rax
400545: cmp $0x3, %rax
400549: jle 400560
40054b: mov %rax, %rdi
40054e: call 0x400600
```

**对应伪代码**：
```
rax = *(%rsp + 8)    // 读取栈上的参数
if (rax > 3) {       // 注意 jle 对应 <=
    func(rax);
}
```

### 7.3 逆向循环结构

```assembly
400570: mov $0x0, %eax    # 初始化
400575: cmp $0xa, %rax
400579: jge 400590        # 循环结束条件
40057b: add (%rbx,%rax,8), %rdx
40057f: add $0x1, %rax
400583: jmp 400575
```

**对应 C 代码**：
```c
for (int i = 0; i < 10; i++) {
    sum += array[i];
}
```

### 7.4 常见陷阱

| 陷阱                   | 解决方案                                          |
| ---------------------- | ------------------------------------------------- |
| `lea` 被误认为内存访问 | 记住 `lea` 只计算地址，不解引用                   |
| `cmp` 方向搞反         | 条件针对第二个操作数                              |
| 无符号与有符号混淆     | `ja`/`jb` 用于无符号，`jg`/`jl` 用于有符号        |
| 寄存器大小混淆         | `%rax`（64位），`%eax`（低32位），`%ax`（低16位） |

1. **识别数据结构**：分析寻址模式中的比例因子（1、2、4、8）

3. **追踪数据流**：确定如何从输入映射到数组索引
4. **选择有效元素**：找到数组中满足条件的元素值

```bash
echo "password" > input.txt

# 运行炸弹
./bomb input.txt

# 在 GDB 中使用
(gdb) r < input.txt
```

### 查看字符串存储
```bash
# 在字符串比较前设置断点
(gdb) b *0x400540    # 替换为实际地址

# 查看输入的字符串
(gdb) x /s $rdi      # 第一个参数通常是指向输入字符串的指针
```

### 检查内存中的静态数据
```bash
(gdb) x /s 0x402000  # 查看字符串常量
(gdb) x /8wx 0x402000 # 查看整型数组
```

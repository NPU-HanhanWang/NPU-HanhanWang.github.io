# 计算机系统基础：位、字节与整数

## 1. 信息表示：从模拟到数字

### 1.1 模拟计算机的局限

在数字计算机之前，模拟计算机通过连续电压表示数值。例如，电压加法器电路通过可调电阻改变电压输出。
**问题**：噪声、元件精度限制使得难以精确测量连续值。

### 1.2 二进制革命

- 我们只区分两个状态：**高电压（On）** 和 **低电压（Off）**
- 两个状态对应二进制数字：`0` 和 `1`
- 优势：容忍噪声、易于复制、存储和传输

## 2. 二进制数与进制转换

### 2.1 二进制表示基础

每个二进制位（bit）代表一个权重，从最低位（LSB）$2^0$ 到最高位（MSB）$2^{w-1}$。

**公式**：
$$B2U(X) = \sum_{i=0}^{w-1} x_i \cdot 2^i$$

示例（3位二进制）：
```
000₂ = 0*4 + 0*2 + 0*1 = 0₁₀
001₂ = 0*4 + 0*2 + 1*1 = 1₁₀
010₂ = 0*4 + 1*2 + 0*1 = 2₁₀
011₂ = 0*4 + 1*2 + 1*1 = 3₁₀
100₂ = 1*4 + 0*2 + 0*1 = 4₁₀
```

### 2.2 十六进制（Hexadecimal）

- **为什么用十六进制**：二进制写起来太长，十六进制每位对应4个bit，易于与二进制互转。
- **符号**：`0`-`9` 和 `A`-`F`（`A`=10, `B`=11, `C`=12, `D`=13, `E`=14, `F`=15）
- **C语言前缀**：`0x` 或 `0X`

#### 十六进制 ↔ 二进制对照表

| Hex | Decimal | Binary | Hex | Decimal | Binary |
| --- | ------- | ------ | --- | ------- | ------ |
| 0   | 0       | 0000   | 8   | 8       | 1000   |
| 1   | 1       | 0001   | 9   | 9       | 1001   |
| 2   | 2       | 0010   | A   | 10      | 1010   |
| 3   | 3       | 0011   | B   | 11      | 1011   |
| 4   | 4       | 0100   | C   | 12      | 1100   |
| 5   | 5       | 0101   | D   | 13      | 1101   |
| 6   | 6       | 0110   | E   | 14      | 1110   |
| 7   | 7       | 0111   | F   | 15      | 1111   |

示例：`0x1A2B` = $1 \times 16^3 + 10 \times 16^2 + 2 \times 16^1 + 11 \times 16^0 = 6699_{10}$

## 3. 位级运算

### 3.1 布尔代数与位运算

| 运算 | 符号 | 规则                 |
| ---- | ---- | -------------------- |
| AND  | `&`  | 1 当且仅当两位都为 1 |
| OR   | `\|` | 1 如果至少一位为 1   |
| NOT  | `~`  | 取反（0变1，1变0）   |
| XOR  | `^`  | 1 当两位不同（异或） |

**性质**：所有布尔代数性质（交换律、结合律、分配律、德摩根定律）对位运算成立。

### 3.2 位运算在C语言中的应用

```c
// 对char类型进行位运算
unsigned char a = 0x41;   // 01000001₂
unsigned char b = 0x55;   // 01010101₂
unsigned char c = a & b;  // 01000001₂ = 0x41
unsigned char d = a | b;  // 01111101₂ = 0x7D
unsigned char e = a ^ b;  // 00010100₂ = 0x14
unsigned char f = ~a;     // 10111110₂ = 0xBE
```

### 3.3 集合表示（位向量的应用）

用宽度为 $w$ 的位向量表示集合 $\{0, 1, ..., w-1\}$：
- 位 $j = 1$ 表示元素 $j$ 在集合中

示例（$w=8$）：
```
位向量：01101001₂ → 集合 {0, 3, 5, 6}
位向量：01010101₂ → 集合 {0, 2, 4, 6}
& 运算 → 01000001₂ → 交集 {0, 6}
| 运算 → 01111101₂ → 并集 {0,2,3,4,5,6}
^ 运算 → 00111100₂ → 对称差 {2,3,4,5}
~ 运算 → 10101010₂ → 补集 {1,3,5,7}
```

### 3.4 逻辑运算符 vs 位运算符（常见陷阱）

| 运算符              | 类型     | 返回值   | 短路求值             |
| ------------------- | -------- | -------- | -------------------- |
| `&`, `\|`, `~`, `^` | 位运算   | 按位结果 | 否                   |
| `&&`, `\|\|`, `!`   | 逻辑运算 | 0 或 1   | 是（`&&` 和 `\|\|`） |

```c
int x = 0x41;   // 非零
int y = 0x00;   // 零
int a = !x;     // 0
int b = !!x;    // 1
int c = x && y; // 0（短路：y 不会被求值）
int d = x & y;  // 0（位运算，不短路）
// 常见错误：if (x & y) 与 if (x && y) 完全不同
```

### 3.5 移位运算

#### 左移 `<<`
- 将位向量左移 $y$ 位，右边补0，左边丢弃超出部分
- 效果：$x << y = x \times 2^y$（无符号或有符号非负数）

#### 右移 `>>`
- **逻辑右移**：左边补0，适用于无符号数
- **算术右移**：左边补符号位（MSB），适用于有符号数（保持负数符号）

示例（8位）：
```
正数：01100010₂ (98)
<< 3 → 00010000₂ (16)    [98*8=784，截断后为16]
>> 2 (逻辑) → 00011000₂ (24)
>> 2 (算术) → 00011000₂ (24)

负数：10100010₂ (-94，假设补码)
<< 3 → 00010000₂ (16)
>> 2 (逻辑) → 00101000₂ (40)   // 错误：负数变正
>> 2 (算术) → 11101000₂ (-24)  // 正确：保持符号
```

**未定义行为**：移位位数 $\ge$ 字长（如 32 位 int 右移 32 位）或为负数。

## 4. 整数的二进制编码

### 4.1 无符号整数（Unsigned）

- 所有位都表示数值
- 范围：$[0, 2^w - 1]$

### 4.2 有符号整数：补码（Two's Complement）

**为什么用补码**？
简单符号位（最高位为符号，其余为数值）有两大问题：
- 存在 `+0` 和 `-0` 两种零
- 加法不能直接使用二进制加法（`1 + (-1)` 不等于0）

**补码公式**：
$$B2T(X) = -x_{w-1} \cdot 2^{w-1} + \sum_{i=0}^{w-2} x_i \cdot 2^i$$

- 最高位权重为 **负** 的 $2^{w-1}$
- 所有负数最高位为 1，非负数最高位为 0

#### 示例（4位补码）：
```
二进制    计算                      十进制
0000    0*8 + 0*4 + 0*2 + 0*1 = 0
0001    0*8 + 0*4 + 0*2 + 1*1 = 1
...
0111    0*8 + 1*4 + 1*2 + 1*1 = 7
1000   -1*8 + 0*4 + 0*2 + 0*1 = -8
1001   -1*8 + 0*4 + 0*2 + 1*1 = -7
...
1111   -1*8 + 1*4 + 1*2 + 1*1 = -1
```

**特点**：
- 范围：$[-2^{w-1}, 2^{w-1}-1]$
- 唯一表示零，没有 `-0`
- 加法与无符号相同的位级行为，方便硬件实现

### 4.3 补码的求负（求 $-x$）

**规则**：按位取反，再加1
$$-x = \sim x + 1$$

**证明**（以4位为例）：
```
x = 0101₂ (5)
~x = 1010₂ (按位取反)
~x + 1 = 1011₂
验证：0101 + 1011 = 10000₂ → 截断为 0000₂，正确
```

**特殊情况**：`TMin`（$-2^{w-1}$）的负数为自身，因为正数范围不够表示。例如 4 位 `TMin = -8 (1000₂)`，`~1000+1 = 0111+1=1000` 仍是 -8。

### 4.4 数值范围（以16位为例）

| 类型 | 十进制 | 十六进制 | 二进制            |
| ---- | ------ | -------- | ----------------- |
| UMax | 65535  | 0xFFFF   | 11111111 11111111 |
| TMax | 32767  | 0x7FFF   | 01111111 11111111 |
| TMin | -32768 | 0x8000   | 10000000 00000000 |
| -1   | -1     | 0xFFFF   | 11111111 11111111 |
| 0    | 0      | 0x0000   | 00000000 00000000 |

## 5. 无符号与有符号的转换

### 5.1 转换规则

- **位模式不变**，仅重新解释含义
- 有符号转无符号：负数变为大的正数（$x + 2^w$）
- 无符号转有符号：大于 $2^{w-1}-1$ 的数变为负数（$x - 2^w$）

#### 映射关系（4位）
| Bits | Signed | Unsigned |
| ---- | ------ | -------- |
| 0000 | 0      | 0        |
| 0001 | 1      | 1        |
| ...  | ...    | ...      |
| 0111 | 7      | 7        |
| 1000 | -8     | 8        |
| 1001 | -7     | 9        |
| 1010 | -6     | 10       |
| 1011 | -5     | 11       |
| 1100 | -4     | 12       |
| 1101 | -3     | 13       |
| 1110 | -2     | 14       |
| 1111 | -1     | 15       |

### 5.2 C语言中的转换

- **显式转换**：`(unsigned) x` 或 `(int) u`
- **隐式转换**：赋值、函数参数、混合类型表达式

```c
int x = -1;
unsigned u = 0xFFFFFFFF;
printf("%d\n", (int) u);  // -1
printf("%u\n", (unsigned) x); // 4294967295
```

### 5.3 常见陷阱：混合类型比较

当有符号和无符号混合出现在表达式中时，**有符号值会被隐式转换为无符号**！

```c
-1 < 0U        // 假！因为 -1 转为 0xFFFFFFFF，0xFFFFFFFF < 0 为假
2147483647U > -2147483647-1  // 取决于具体值，小心
```

<details>
<summary>📝 练习题：判断下列表达式真假（w=32）</summary>

| 表达式                        | 结果   | 解释               |
| ----------------------------- | ------ | ------------------ |
| `0 == 0U`                     | 真     | 值相等             |
| `-1 < 0`                      | 真     | 都是 signed        |
| `-1 < 0U`                     | **假** | -1 转为 4294967295 |
| `2147483647 > -2147483647-1`  | 真     | 两边 signed        |
| `2147483647U > -2147483647-1` | 假     | 右边转为 unsigned  |
| `-1 > -2`                     | 真     |                    |
| `(unsigned)-1 > -2`           | 假     |                    |

</details>

## 6. 扩展与截断

### 6.1 符号扩展（Sign Extension）

**任务**：将 $w$ 位有符号数转换为 $w+k$ 位，保持值不变。

**规则**：复制原最高位（符号位）到新增的高位。

示例（8位 → 16位）：
```
int8_t su = -1;      // 0xFF (11111111₂)
int16_t bu = su;     // 0xFFFF (11111111 11111111₂)
```

证明：补码符号扩展不改变数值。

<details>
<summary>🔍 数学原理</summary>

对于负数的补码表示 $[x_{w-1}, x_{w-2}, ..., x_0]$，扩展一位符号位后，新最高位 $x_{w-1}$ 重复。
原值：$-x_{w-1}2^{w-1} + \sum_{i=0}^{w-2} x_i 2^i$
新值：$-x_{w-1}2^{w} + x_{w-1}2^{w-1} + \sum_{i=0}^{w-2} x_i 2^i = -x_{w-1}2^{w-1} + \sum_{i=0}^{w-2} x_i 2^i$ 相同。
</details>

### 6.2 零扩展（Zero Extension）

**用于无符号数**：高位补0，保持数值。

```c
uint8_t su = 1;       // 0x01
uint16_t bu = su;     // 0x0001
```

### 6.3 截断（Truncation）

**规则**：直接丢弃高位（保留低 $w$ 位）。
- 对无符号：相当于 $\mod 2^w$
- 对有符号：位模式不变后再解释为补码，等价于模 $2^w$ 后再映射到有符号范围

```c
uint16_t bu = 1000;   // 0x03E8
uint8_t su = bu;      // 0xE8 = 232 （1000 mod 256 = 232）
```

```c
uint16_t bu = 0x8000; // 32768
uint8_t su = bu;      // 0x00 = 0 （32768 mod 256 = 0）
```

## 7. 整数算术运算

### 7.1 无符号加法（模运算）

$$UAdd_w(u, v) = (u + v) \mod 2^w$$

- 当 $u + v \ge 2^w$ 时发生**溢出**，结果减去 $2^w$
- 检测溢出：$s = u + v$，溢出当且仅当 $s < u$（或 $s < v$）

示例（8位无符号）：
```
unsigned char u = 223;   // 0xDF (11011111₂)
unsigned char v = 213;   // 0xD5 (11010101₂)
unsigned char s = u + v; // 223+213=436，436 mod 256 = 180 (0xB4)
实际计算：11011111 + 11010101 = 1 10110100 → 截断为 10110100₂ = 180
```

### 7.2 补码加法

**位级行为与无符号加法完全相同**，只是最后将结果解释为补码。

- 正溢出：结果 $< -2^{w-1}$（实际太大正数变负数）
- 负溢出：结果 $\ge 2^{w-1}$（实际太小负数变正数）

```
4位补码示例：5 (0101) + 3 (0011) = 8 (1000) → 实际 -8（正溢出）
-8 (1000) + -8 (1000) = 10000 → 截断 0000 = 0（负溢出？实际 -16 转 0）
```

### 7.3 可视化：补码加法环绕

数轴（4位补码，范围 -8 到 7）：

```
        负溢出区                    正溢出区
    ←─────────┐                ┌─────────→
     -8  -7 ... -1  0  1 ... 6  7
        ↑                      ↑
    加法结果<-8时         加法结果>7时
    绕到正数区             绕到负数区
```

### 7.4 乘法

- 精确乘积最多需要 $2w$ 位
- C语言中乘法结果截断到 $w$ 位
- 无符号/有符号：低位相同，高位解释不同

```c
unsigned x = 0xFFFFFFFFU;  // 2^32 - 1
unsigned y = 0xFFFFFFFFU;
unsigned z = x * y;        // 结果 = 1（因为 (2^32-1)^2 mod 2^32 = 1）
```

### 7.5 乘以常数优化：用移位和加法

编译器自动将乘以常数优化为移位和加减组合，因为移位比乘法快得多。

- `x << k` 等价于 $x \times 2^k$（无符号或有符号非负）
- 一般乘法：例如 $x \times 14 = (x << 3) + (x << 2) + (x << 1)$
- 更高效：$x \times 14 = (x << 4) - (x << 1)$

```c
int mul_by_24(int x) {
    // 编译器可能生成： (x << 5) - (x << 3)
    return x * 24;
}
```

### 7.6 除以 2 的幂（无符号与有符号）

#### 无符号除法（逻辑右移）
$$x / 2^k = x \gg k \quad (\text{逻辑右移})$$

```c
unsigned u = 4;   // 0100
u >> 1;           // 0010 = 2 ✅
```

#### 有符号除法（算术右移）
- **正数**：`x >> k` 向下取整（floor）
- **负数**：算术右移也是向下取整，但 C 语言要求向零取整

示例（-5 / 2）：
```
-5 的补码（4位）：1011
算术右移1位：1101₂ = -3（向下取整）
但 C 语言 -5/2 期望 -2（向零取整）
```

**解决方法：添加偏置（bias）**
对于除以 $2^k$，向零取整的公式：
$$(x + (1 \ll k) - 1) \gg k \quad \text{当 } x < 0$$

```c
int divide_by_power2(int x, int k) {
    // 适用于负数向零取整
    int bias = (x >> 31) & ((1 << k) - 1);  // x<0 时 bias=2^k-1，否则0
    return (x + bias) >> k;
}
```

示例：$x=-5, k=1$，$bias = ( -5 >> 31 ) \& 1 = 0xFFFFFFFF \& 1 = 1$，$(-5+1)>>1 = -4>>1 = -2$ ✅

## 8. 字节序（Byte Ordering）

### 8.1 大端序 vs 小端序

对于多字节整数（如 4 字节的 `0x01234567` 存储在地址 `0x100`）：

| 地址  | 大端序（Big Endian）   | 小端序（Little Endian） |
| ----- | ---------------------- | ----------------------- |
| 0x100 | `0x01`（最高有效字节） | `0x67`（最低有效字节）  |
| 0x101 | `0x23`                 | `0x45`                  |
| 0x102 | `0x45`                 | `0x23`                  |
| 0x103 | `0x67`（最低有效字节） | `0x01`（最高有效字节）  |

### 8.2 常见平台

- **大端**：网络协议（TCP/IP）、SPARC、旧 Mac（PowerPC）
- **小端**：x86、ARM（大多数手机、PC）

### 8.3 字节序的影响

- **不影响**：单字节字符、字符串（因为每个字符独立）
- **影响**：二进制数据跨平台传输、类型转换（如 `int` 转 `char*` 读取）

### 8.4 检查本机字节序的代码

```c
#include <stdio.h>
int main() {
    int x = 0x12345678;
    char *c = (char*) &x;
    if (*c == 0x78)
        printf("Little Endian\n");
    else
        printf("Big Endian\n");
    return 0;
}
```

### 8.5 反汇编中的字节序

当查看机器码反汇编时，指令的字节序与数据相同。例如，x86 小端序中，指令 `mov $0x12345678, %eax` 的机器码可能会显示为 `78 56 34 12`。

## 9. 总结速查表

| 操作      | 无符号                  | 补码有符号                                |
| --------- | ----------------------- | ----------------------------------------- |
| 范围      | $[0, 2^w-1]$            | $[-2^{w-1}, 2^{w-1}-1]$                   |
| 求负      | 不适用（无负数）        | `~x + 1`                                  |
| 加法      | 模 $2^w$                | 同无符号位级，解释不同                    |
| 乘法      | 截断低位 $w$ 位         | 同左                                      |
| 左移 `<<` | 乘以 $2^k$              | 乘以 $2^k$（小心溢出）                    |
| 右移 `>>` | 逻辑右移（补0）         | 算术右移（补符号位）                      |
| 扩展      | 零扩展                  | 符号扩展                                  |
| 截断      | 模 $2^w$                | 模 $2^w$ 后补码解释                       |
| 转换规则  | 有符号→无符号：加 $2^w$ | 无符号→有符号：减 $2^w$（若 $x>2^{w-1}$） |

### 常见易错点

1. **`-1 < 0U`** 永远为假（因为 `-1` 转成无符号最大值）
2. **`>>` 对有符号数是算术右移**（C标准实现定义，但几乎所有编译器都采用算术右移）
3. **补码的 `TMin` 求负得到自身**，可能导致 `abs(TMin)` 溢出
4. **混合类型运算时，有符号会转为无符号**，务必小心比较和算术表达式
5. **移位位数不能等于或超过字长**，否则行为未定义

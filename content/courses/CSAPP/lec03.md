# 计算机系统基础：机器级编程（汇编基础）

## 1. 抽象层次：从C语言到硬件

计算机系统采用分层抽象，不同层次的程序员看到不同的“机器”：

```
┌─────────────────────────────────────────────────────────────┐
│  C 程序员                                                    │
│  int main() { int i, n=10, t1=0, t2=1, nxt;                 │
│    for (i=1; i<=n; ++i) { printf("%d, ", t1);               │
│      nxt = t1 + t2; t1 = t2; t2 = nxt; } return 0; }        │
└─────────────────────────────────────────────────────────────┘
                              ↓ 编译
┌─────────────────────────────────────────────────────────────┐
│  汇编程序员                                                  │
│  sumstore:  pushq  %rbx                                     │
│            movq   %rdx, %rbx                                │
│            call   plus                                      │
│            movq   %rax, (%rbx)                              │
│            popq   %rbx                                      │
│            ret                                              │
└─────────────────────────────────────────────────────────────┘
                              ↓ 汇编/链接
┌─────────────────────────────────────────────────────────────┐
│  机器码（字节序列）                                           │
│  0x53 0x48 0x89 0xd3 0xe8 ...                               │
└─────────────────────────────────────────────────────────────┘
                              ↓ 执行
┌─────────────────────────────────────────────────────────────┐
│  计算机设计者（微架构）                                        │
│  逻辑门、时钟、电路布局...                                     │
└─────────────────────────────────────────────────────────────┘
```

**关键概念**：
- **ISA（指令集架构）**：程序员需要了解的处理器设计部分（指令集、寄存器等）
- **微架构**：ISA的具体硬件实现（缓存大小、主频等）

## 2. x86-64 整数寄存器

x86-64 提供 **16 个 64 位通用寄存器**，历史沿革：
- 16位时代：`ax`, `bx`, `cx`, `dx`, `si`, `di`, `sp`, `bp`
- 32位时代（80386）：扩展为 `eax`, `ebx`, ...（`e` 表示 extended）
- 64位时代（AMD x86-64）：扩展为 `rax`, `rbx`, ...（以及新增 `r8`-`r15`）

### 2.1 寄存器名称与部分访问

每个寄存器可以访问不同大小的“子区域”：

```
63                31 15 7 0
┌──────────────────┬──┬──┬──┐
│        rax       │  │  │  │  64位 (qword)
├──────────────────┴──┴──┴──┤
│         eax               │  32位 (dword)
├──────────────────┬──┬──┬──┤
│         ax       │  │  │  16位 (word)
├──────────┬───────┴──┴──┴──┤
│    ah    │        al      │  8位高位和低位 (byte)
└──────────┴────────────────┘
```

**常用寄存器列表**：

| 64位   | 32位   | 16位   | 8位（低） | 用途约定          |
| ------ | ------ | ------ | --------- | ----------------- |
| `%rax` | `%eax` | `%ax`  | `%al`     | 返回值            |
| `%rbx` | `%ebx` | `%bx`  | `%bl`     | 被调用者保存      |
| `%rcx` | `%ecx` | `%cx`  | `%cl`     | 第4个参数         |
| `%rdx` | `%edx` | `%dx`  | `%dl`     | 第3个参数         |
| `%rsi` | `%esi` | `%si`  | `%sil`    | 第2个参数         |
| `%rdi` | `%edi` | `%di`  | `%dil`    | 第1个参数         |
| `%rsp` | `%esp` | `%sp`  | `%spl`    | 栈指针            |
| `%rbp` | `%ebp` | `%bp`  | `%bpl`    | 帧指针（可选）    |
| `%r8`  | `%r8d` | `%r8w` | `%r8b`    | 第5个参数         |
| ...    | ...    | ...    | ...       | 依次类推至 `%r15` |

## 3. 数据传送指令 `mov`

### 3.1 基本语法

```assembly
movq 源, 目的    # 传送 8 字节 (quad word)
movl 源, 目的    # 传送 4 字节 (double word)
movw 源, 目的    # 传送 2 字节 (word)
movb 源, 目的    # 传送 1 字节 (byte)
```

### 3.2 操作数类型

| 类型   | 语法         | 示例                | 含义                 |
| ------ | ------------ | ------------------- | -------------------- |
| 立即数 | `$Imm`       | `$0x1F`, `$-5`      | 常量值               |
| 寄存器 | `%reg`       | `%rax`, `%r8d`      | 寄存器内容           |
| 内存   | `(Rb)`       | `(%rdi)`            | `Mem[Rb]`            |
| 内存   | `D(Rb)`      | `0x8(%rsp)`         | `Mem[Rb + D]`        |
| 内存   | `(Rb,Ri)`    | `(%rdi,%rsi)`       | `Mem[Rb + Ri]`       |
| 内存   | `D(Rb,Ri,S)` | `0x80(%rdi,%rsi,4)` | `Mem[Rb + S*Ri + D]` |

**重要约束**：**不允许内存到内存的直接传送**，必须经过寄存器中转。

### 3.3 内存寻址模式详解

通用形式：**`D(Rb, Ri, S)`**
计算公式：`Mem[ Reg[Rb] + S * Reg[Ri] + D ]`

- `D`：常量位移（1, 2 或 4 字节）
- `Rb`：基址寄存器（任意 16 个通用寄存器）
- `Ri`：索引寄存器（除 `%rsp` 外的任意寄存器）
- `S`：比例因子（1, 2, 4, 或 8）——对应常见数据类型大小

**寻址示例**（假设 `%rdx = 0xF000`, `%rcx = 0x0100`）：

| 表达式            | 地址计算           | 结果地址  |
| ----------------- | ------------------ | --------- |
| `0x8(%rdx)`       | `0xF000 + 8`       | `0xF008`  |
| `(%rdx, %rcx)`    | `0xF000 + 0x100`   | `0xF100`  |
| `(%rdx, %rcx, 4)` | `0xF000 + 4*0x100` | `0xF400`  |
| `0x80(, %rdx, 2)` | `2*0xF000 + 0x80`  | `0x1E080` |

### 3.4 操作数组合合法性

```
┌──────────┬───────────┬───────────┬───────────┐
│  源      │   目的    │ 是否允许  │ 示例      │
├──────────┼───────────┼───────────┼───────────┤
│ 立即数   │ 寄存器    │ ✅        │ movq $5, %rax │
│ 立即数   │ 内存      │ ✅        │ movq $5, (%rdi) │
│ 寄存器   │ 寄存器    │ ✅        │ movq %rax, %rbx │
│ 寄存器   │ 内存      │ ✅        │ movq %rax, (%rdi) │
│ 内存     │ 寄存器    │ ✅        │ movq (%rdi), %rax │
│ 内存     │ 内存      │ ❌        │ 不允许！     │
└──────────┴───────────┴───────────┴───────────┘
```

### 3.5 示例：交换函数

```c
void swap(long *xp, long *yp) {
    long t0 = *xp;
    long t1 = *yp;
    *xp = t1;
    *yp = t0;
}
```

对应的汇编：

```assembly
swap:
    movq    (%rdi), %rax    # t0 = *xp   (xp 在 %rdi)
    movq    (%rsi), %rdx    # t1 = *yp   (yp 在 %rsi)
    movq    %rdx, (%rdi)    # *xp = t1
    movq    %rax, (%rsi)    # *yp = t0
    ret
```

**内存变化示意**（假设初始内存内容）：

```
初始状态:
%rdi = 0x120, %rsi = 0x100
地址 0x120: 0x123, 地址 0x100: 0x456

执行 movq (%rdi), %rax  → %rax = 0x123
执行 movq (%rsi), %rdx  → %rdx = 0x456
执行 movq %rdx, (%rdi)  → 地址 0x120 变为 0x456
执行 movq %rax, (%rsi)  → 地址 0x100 变为 0x123
结果：两个内存位置的值互换。
```

## 4. 地址计算指令 `leaq`

`leaq`（Load Effective Address）是 **不访问内存** 的地址计算指令。

```assembly
leaq 源, 目的    # 目的 = 源的地址（而非地址中的值）
```

### 4.1 两种用途

1. **计算指针地址**（类似 C 语言的 `&`）
2. **高效算术运算**（利用寻址模式进行乘法和加法）

### 4.2 算术运算示例

```c
long m12(long x) {
    return x * 12;
}
```

编译生成的汇编：

```assembly
m12:
    leaq    (%rdi, %rdi, 2), %rax   # t = x + 2*x = 3*x
    salq    $2, %rax                # t << 2 = 12*x
    ret
```

**解释**：
- `leaq` 没有内存访问，只是计算 `%rdi + 2*%rdi = 3*x`
- `salq $2` 左移 2 位相当于乘以 4，最终 `3*x * 4 = 12*x`

### 4.3 更复杂的算术表达式示例

```c
long arith(long x, long y, long z) {
    long t1 = x + y;
    long t2 = z + t1;
    long t3 = x + 4;
    long t4 = y * 48;
    long t5 = t3 + t4;
    long rval = t2 * t5;
    return rval;
}
```

汇编代码及寄存器映射：

```assembly
arith:
    leaq    (%rdi, %rsi), %rax      # t1 = x + y
    addq    %rdx, %rax              # t2 = z + t1
    leaq    (%rsi, %rsi, 2), %rdx   # 3*y
    salq    $4, %rdx                # t4 = (3*y) << 4 = 48*y
    leaq    4(%rdi, %rdx), %rcx     # t5 = x + t4 + 4
    imulq   %rcx, %rax              # rval = t2 * t5
    ret
```

| 寄存器 | 用途                   |
| ------ | ---------------------- |
| `%rdi` | 参数 `x`               |
| `%rsi` | 参数 `y`               |
| `%rdx` | 参数 `z`，后用于 `t4`  |
| `%rax` | `t1`, `t2`, 最终返回值 |
| `%rcx` | `t5`                   |

## 5. 算术与逻辑运算指令

### 5.1 双操作数指令

格式：`操作码 源, 目的`  →  `目的 = 目的 op 源`

| 指令                         | 含义                    | C 等价                   |
| ---------------------------- | ----------------------- | ------------------------ |
| `addq 源, 目的`              | 目的 += 源              | `dest += src`            |
| `subq 源, 目的`              | 目的 -= 源              | `dest -= src`            |
| `imulq 源, 目的`             | 目的 *= 源（有符号）    | `dest *= src`            |
| `salq 源, 目的`（或 `shlq`） | 目的 <<= 源（左移）     | `dest <<= src`           |
| `sarq 源, 目的`              | 目的 >>= 源（算术右移） | `dest >>= src`（有符号） |
| `shrq 源, 目的`              | 目的 >>= 源（逻辑右移） | `dest >>= src`（无符号） |
| `xorq 源, 目的`              | 目的 ^= 源              | `dest ^= src`            |
| `andq 源, 目的`              | 目的 &= 源              | `dest &= src`            |
| `orq 源, 目的`               | 目的 \|= 源             | `dest \|= src`           |

### 5.2 单操作数指令

| 指令        | 含义         |
| ----------- | ------------ |
| `incq 目的` | 目的 += 1    |
| `decq 目的` | 目的 -= 1    |
| `negq 目的` | 目的 = -目的 |
| `notq 目的` | 目的 = ~目的 |

## 6. 条件码与条件跳转

### 6.1 条件码寄存器（Condition Codes）

x86-64 维护一组 **1 位条件码**，记录最近算术/逻辑运算的“副作用”：

| 条件码 | 含义     | 设置条件                             |
| ------ | -------- | ------------------------------------ |
| `CF`   | 进位标志 | 无符号溢出（加法进位或减法借位）     |
| `ZF`   | 零标志   | 结果为 0                             |
| `SF`   | 符号标志 | 结果为负数（最高位为1）              |
| `OF`   | 溢出标志 | 有符号溢出（正+正得负 或 负+负得正） |

大多数运算指令（`add`, `sub`, `imul`, `and`, `or`, `xor`, `sal`, `sar`, `shl`, `shr`）会设置条件码。
`leaq` 和 `mov` **不会** 修改条件码。

### 6.2 比较指令 `cmp`

```assembly
cmpq 源2, 源1    # 计算 源1 - 源2，设置条件码，不修改寄存器
```

- 等价于 `sub` 但 **不保存结果**
- 用于条件判断前设置 `CF`、`ZF`、`SF`、`OF`

### 6.3 测试指令 `test`

```assembly
testq 源2, 源1   # 计算 源1 & 源2，设置 SF 和 ZF，不修改寄存器
```

常见用法：
```assembly
testq %rax, %rax   # 检查 %rax 是否为 0
je    label        # 如果为零则跳转
```

### 6.4 条件跳转指令 `jX`

| 指令  | 条件           | 含义（有符号） | 指令  | 条件      | 含义（无符号） |
| ----- | -------------- | -------------- | ----- | --------- | -------------- |
| `je`  | ZF=1           | 相等 / 为零    | `je`  | ZF=1      | 相等           |
| `jne` | ZF=0           | 不相等 / 非零  | `jne` | ZF=0      | 不相等         |
| `jg`  | ~(SF^OF) & ~ZF | 大于           | `ja`  | ~CF & ~ZF | 高于           |
| `jge` | ~(SF^OF)       | 大于等于       | `jae` | ~CF       | 高于等于       |
| `jl`  | SF^OF          | 小于           | `jb`  | CF=1      | 低于           |
| `jle` | (SF^OF) \| ZF  | 小于等于       | `jbe` | CF \| ZF  | 低于等于       |
| `js`  | SF=1           | 负数           | -     | -         | -              |
| `jns` | SF=0           | 非负数         | -     | -         | -              |

<details>
<summary>🔍 为什么有符号和无符号使用不同的跳转条件？</summary>

`cmp` 指令根据两种不同的溢出定义设置条件码。对于有符号数，`OF` 指示符号溢出；对于无符号数，`CF` 指示进位/借位。
因此 `jg`（有符号大于）检查 `SF==OF && ZF==0`，而 `ja`（无符号高于）检查 `CF==0 && ZF==0`。
**注意**：C 语言中的 `>` 会根据操作数类型选择正确的汇编指令。
</details>

### 6.5 条件执行示例

```c
void decision(int x) {
    if (x) {
        op1();
    } else {
        op2();
    }
}
```

对应的汇编：

```assembly
decision:
    testl   %edi, %edi      # 测试 x（参数在 %edi）是否为 0
    je      .L2             # 如果 x == 0，跳转到 else 分支
    call    op1             # 调用 op1()
    jmp     .L1             # 跳转到函数结尾
.L2:
    call    op2             # 调用 op2()
.L1:
    ret
```

## 7. 从 C 到机器码的完整流程

### 7.1 编译步骤

```
源文件 (p1.c, p2.c)
       ↓ gcc -Og -S
汇编文件 (p1.s, p2.s)
       ↓ as (汇编器)
目标文件 (p1.o, p2.o)   ← 二进制机器码，但符号未解析
       ↓ ld (链接器)
可执行文件 (p)           ← 完整机器码
```

### 7.2 查看汇编代码

```bash
gcc -Og -S sum.c          # 生成 sum.s 汇编文件
gcc -Og -c sum.c          # 生成 sum.o 目标文件
objdump -d sum.o          # 反汇编目标文件
```

### 7.3 示例：`sumstore` 函数的完整过程

**C 代码** (`sum.c`)：
```c
long plus(long x, long y);

void sumstore(long x, long y, long *dest) {
    long t = plus(x, y);
    *dest = t;
}
```

**生成的汇编**（`gcc -Og -S`，简化后）：

```assembly
sumstore:
    pushq   %rbx            # 保存 %rbx
    movq    %rdx, %rbx      # 将 dest 保存到 %rbx（因为调用 plus 会覆盖 %rdx）
    call    plus            # 调用 plus(x, y)，结果在 %rax
    movq    %rax, (%rbx)    # *dest = 返回值
    popq    %rbx            # 恢复 %rbx
    ret
```

**机器码**（反汇编 `objdump -d`）：

```
0000000000400595 <sumstore>:
  400595: 53                    push   %rbx
  400596: 48 89 d3              mov    %rdx, %rbx
  400599: e8 f2 ff ff ff        callq  400590 <plus>
  40059e: 48 89 03              mov    %rax, (%rbx)
  4005a1: 5b                    pop    %rbx
  4005a2: c3                    retq
```

**字节序列**（`x/14xb sumstore` 在 gdb 中）：
```
0x53 0x48 0x89 0xd3 0xe8 0xf2 0xff 0xff 0xff 0x48 0x89 0x03 0x5b 0xc3
```

## 9. x86 处理器历史与现状

### 9.1 发展里程碑

| 年代 | 处理器     | 特点                               |
| ---- | ---------- | ---------------------------------- |
| 1978 | 8086       | 16位，1MB地址空间，IBM PC基础      |
| 1985 | 386        | 32位（IA32），平坦寻址，可运行Unix |
| 2004 | Pentium 4E | 首次支持64位（x86-64）             |
| 2006 | Core 2     | 首个多核 x86 处理器                |
| 2008 | Core i7    | 4核（如 shark 机器）               |

### 9.2 x86-64 的由来

- **Itanium (IA64)**：Intel 2001 年推出的全新 64 位架构，不兼容 x86，性能不佳
- **AMD64**：AMD 2003 年推出 **x86-64**，在 32 位 x86 上优雅扩展至 64 位
- **Intel 被迫采纳**：2004 年推出 EM64T（几乎与 AMD64 相同），后统称 x86-64

### 9.3 CISC vs RISC

- **CISC**（如 x86）：指令多、长度可变、功能复杂，编译后代码密度高
- **RISC**（如 ARM、RISC-V）：指令精简、定长，易于流水线和低功耗
- 现代 x86 内部将 CISC 指令解码为类似 RISC 的微操作

## 10. 核心知识点速查

### 调用约定（x86-64 System V ABI）

| 参数顺序 | 寄存器 |
| -------- | ------ |
| 第1个    | `%rdi` |
| 第2个    | `%rsi` |
| 第3个    | `%rdx` |
| 第4个    | `%rcx` |
| 第5个    | `%r8`  |
| 第6个    | `%r9`  |
| 返回值   | `%rax` |

### 指令后缀与操作数大小

| 后缀 | 大小（字节） | 示例            |
| ---- | ------------ | --------------- |
| `b`  | 1            | `movb`, `cmpb`  |
| `w`  | 2            | `movw`, `addw`  |
| `l`  | 4            | `movl`, `imull` |
| `q`  | 8            | `movq`, `addq`  |

### 常见陷阱

1. **内存-内存不能直接传送**：`movq (%rdi), (%rsi)` 非法
2. **`leaq` 不访问内存**，只计算地址
3. **条件码受大部分运算指令影响**，`mov` 和 `lea` 除外
4. **`cmp` 和 `test` 不修改目标寄存器**，只设置条件码
5. **无符号与有符号条件跳转不同**：`ja` vs `jg`

判断以下 `movq` 指令是否合法：
1. `movq $0x123, %rax`

3. ❌ 内存 → 内存（不允许）
4. ✅ 寄存器 → 内存（带偏移）
5. ✅ 内存（复杂寻址） → 寄存器

<details>
<summary>📝 问题 2：leaq 计算表达式</summary>

假设 `%rdi = 5`，`%rsi = 7`，计算以下每条 `leaq` 后 `%rax` 的值：
1. `leaq (%rdi, %rsi), %rax`
2. `leaq 8(%rdi, %rsi, 2), %rax`
3. `leaq (,%rsi, 4), %rax`

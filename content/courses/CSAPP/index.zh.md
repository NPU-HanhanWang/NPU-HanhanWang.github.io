# CMU 15-213/14-513/15-513: 计算机系统导论

这份文档是 CMU 15-213/14-513/15-513: 计算机系统导论 (Introduction to Computer Systems, ICS) 2026年春季学期的课程概览（Syllabus/Overview）。该课程被誉为 CMU 计算机科学与计算机工程领域的“核心基石”。
以下是对文档关键信息的深度总结与归纳：

1. 课程核心理念：打破“抽象”的迷思
课程的核心在于教导学生从“抽象”回归“现实”。虽然大多数 CS 课程强调抽象（如 ADT、渐进分析），但本课程强调理解抽象之下的底层实现及其局限性。
五大“现实”定律（The Five Great Realities）：
数据表示的局限性：Ints 不是数学整数，Floats 不是实数（存在溢出、精度丢失问题）。
汇编语言的重要性：理解 C/C++ 代码编译后的机器指令是理解程序执行模型的关键。
内存的非均匀性：内存不仅是有限的，且访问速度不一致（缓存、虚拟内存层级）。
性能常数因子：常数因子（Constant factors）在实际系统中至关重要，不能仅关注渐进复杂度。
I/O 与并发：计算机不仅要计算，还需处理外部输入输出、网络通信及并发操作。

2. 编程实践与调试（C语言）
课程强调通过 C 语言直接操作底层硬件，并指出了常见的编程陷阱：
内存引用错误：C 语言缺乏内存保护，越界访问（Out of bounds）和指针错误会导致隐蔽的 Bug（如幻数 0x40000000 导致的浮点数覆盖问题）。
调试哲学：调试是一个必须通过练习掌握的过程（Process）。课程明确禁止使用 AI 工具（如 GitHub Copilot, ChatGPT）进行代码生成或调试，强调手动使用 gdb、valgrind 等工具定位问题。

3. 学术诚信与 AI 政策（严格！）
文档对学术诚信（Academic Integrity）有极其严格的规定，特别是针对 AI 工具的使用：
严禁 AI 代码生成：禁止使用任何 AI 工具（包括云基、本地或 IDE 内置）生成、补全或调试代码。
严禁网络搜索：禁止在互联网上搜索与作业相关的关键字（如 "213 malloc solution"），甚至禁止搜索课程编号。
版本控制（Git）：必须使用 Git 进行频繁提交（Commit early and often）。合理的提交历史是证明代码原创性的重要依据。
后果严重：一旦作弊（AIV），最低处罚为作业 -100%，最高可导致课程不及格甚至被大学开除。

4. 课程实验体系（Labs）
课程通过 8 个极具挑战性的实验（Labs）贯彻“做中学”（Learning by Doing）的理念，覆盖了从位操作到并发网络编程的广泛领域。


| 实验编号 | 名称              | 核心技术点                 | 目标                                       |
| :------- | :---------------- | :------------------------- | :----------------------------------------- |
| L0       | C Programming Lab | C 语言基础、指针、内存管理 | 复习 C 语言，确保具备基本编程能力          |
| L1       | Data Lab          | 位操作、补码、浮点数编码   | 仅使用位运算符操作整数和浮点数             |
| L2       | Bomb Lab          | x86 汇编、调试             | 逆向工程，通过阅读汇编代码“拆除”二进制炸弹 |
| L3       | Attack Lab        | 缓冲区溢出、Shellcode      | 利用漏洞进行代码注入攻击，理解安全机制     |
| L4       | Cache Lab         | 缓存机制、局部性           | 编写缓存模拟器，并优化矩阵转置代码         |
| L5       | Malloc Lab        | 动态内存分配               | 实现自己的 `malloc`, `free`, `realloc`     |
| L6       | TSH Lab           | 进程控制、Unix 信号        | 实现一个具备作业控制功能的 Unix Shell      |
| L7       | Proxy Lab         | 网络编程、并发             | 实现一个多线程的 Web 代理服务器            |
| L8       | SFS Lab           | 文件系统、并行性           | 扩展一个基本的文件系统，理解同步与性能     |



5. 考核方式与资源
考核构成：
实验 (Labs)：50%（权重最高，按工作量加权）。
书面作业 (Written Assignments)：10%（取最好的 8 次，包含互评环节）。
期中考试：15%。
期末考试：25%。
核心教材：《Computer Systems: A Programmer's Perspective》 (CS:APP3e)，被称为“CSAPP”或“圣经”。
辅助资源：推荐《C Programming Language》(K&R) 作为 C 语言参考，以及一系列“Bootcamp”视频（涵盖 Linux, Git, GDB, Makefile 等）。

6. 关键建议
尽早开始：实验难度大，需要时间去“悟”，且存在 5 天的宽限期（Grace Days），建议留到后期使用。
善用工具：熟练掌握 gdb, valgrind, git 和 Autolab 系统。
寻求帮助：利用好 Office Hours 和 ED 论坛，但讨论仅限于高层设计，禁止代码级抄袭。

# 一、语言模型 (Language Model)

## 1.1 定义与目标

-   **语言模型**：对自然语言中**符号序列**（词或字符）的概率分布进行建模。
-   **核心任务**：给定前面的符号，预测下一个符号的概率。
    <br>
    $$ P(w_1, w_2, \dots, w_T) = \prod_{t=1}^{T} P(w_t \mid w_1, \dots, w_{t-1}) $$
    <br>
-   **应用**：
    1.  **计算序列概率**：评估一个句子的合理性（如语法纠错、机器翻译的候选排序）。
    2.  **生成新序列**：根据给定的"提示词"（prompt）自动续写文本。

## 1.2 词的表示：从独热编码到嵌入向量

### 1.2.1 独热编码 (One-Hot) 的问题

-   **维度灾难**：词汇量 $N$ 通常为几万到几十万，向量极其稀疏。
-   **语义真空**：任意两个不同词的独热向量**正交**，无法表达词与词之间的语义相似性（如"猫"和"狗"的距离与"猫"和"汽车"的距离相同）。
-   **空间利用率低**：$N$ 个词只占用了 $N$ 维空间中 $N$ 个点，体积几乎为零。

### 1.2.2 词嵌入 (Word Embedding)

-   **思想**：将高维独热向量投影到一个低维稠密的实数向量空间（通常 50-1000 维）。
    <br>
    $$ e = P \cdot \text{one-hot}(w) $$
    <br>
    其中 $P$ 是一个 $M \times N$ 的可学习线性变换矩阵。
-   **效果**：嵌入后的向量空间具有**语义结构**。语义相近的词在向量空间中距离更近，甚至能捕捉到类似 $ \text{国王} - \text{王后} \approx \text{男人} - \text{女人} $ 这样的类比关系。

    ```
        词嵌入空间的二维PCA投影（示意图）：

               ^
               |  猫
               |   狗
               |  老虎
               |
               |        汽车
               |            火车
               |                飞机
               +------------------------>
    ```

-   **如何学习**：嵌入矩阵 $P$ 通常作为神经网络的第一层（`nn.Embedding`），随语言模型任务一同训练。

<details>
<summary><b>📊 常用嵌入方法对比</b></summary>

| 方法                        | 描述                     | 优点               | 缺点                       |
| :-------------------------- | :----------------------- | :----------------- | :------------------------- |
| **独热编码**                | 一个1，其余全0           | 简单，无先验假设   | 维度高，无语义信息         |
| **Word2Vec (Skip-gram)**    | 用当前词预测上下文       | 训练快，效果好     | 静态嵌入，无法处理一词多义 |
| **GloVe**                   | 基于全局词共现矩阵分解   | 利用了全局统计信息 | 静态嵌入                   |
| **上下文嵌入 (ELMo, BERT)** | 根据上下文动态生成词表示 | 能处理一词多义     | 计算量大                   |
</details>

# 二、序列到序列 (Seq2Seq) 模型

## 2.1 问题定义与架构

-   **任务**：将一个变长输入序列 $X_1 \dots X_N$ 转换为另一个变长输出序列 $Y_1 \dots Y_M$。
-   **典型应用**：机器翻译（英 $\rightarrow$ 德）、文本摘要、语音识别、对话系统、图像描述生成。
-   **核心挑战**：输入和输出序列的长度不同，且没有时间上的对齐关系。

-   **通用架构：编码器-解码器 (Encoder-Decoder)**

    ```
         编码器                      解码器
         (处理输入)                  (生成输出)

         X1 ---> [RNN/LSTM] ---> h1
                                |
         X2 ---> [RNN/LSTM] ---> h2
                                |      Y1 <--- [RNN/LSTM] <--- h'_1
         ...                     |        ^                      ^
                                |        |                      |
         XN ---> [RNN/LSTM] ---> hN      +--- 上下文向量 C ---+--- Y_{t-1}
                                   \                       /
                                    \--> 上下文向量 C (编码器最终状态)
    ```

-   **编码器**：逐个读取输入序列，最终的隐藏状态 $h_N$ 被视为对整个输入序列的**摘要**（上下文向量 $C$）。
-   **解码器**：以 $C$ 作为初始状态，以特殊的 `<SOS>` (Start Of Sentence) 符号作为第一个输入，开始自回归地生成输出序列，直到生成 `<EOS>` (End Of Sentence) 符号为止。

## 2.2 训练与推理

-   **训练**：使用**教师强制 (Teacher Forcing)** 策略。将真实的目标序列（而非模型自己生成的词）作为解码器的输入，计算交叉熵损失。
    <br>
    $$ \text{Loss} = -\sum_{t=1}^{M} \log P(Y_t^{\text{真}} \mid Y_{<t}^{\text{真}}, X) $$
    <br>
-   **推理**：模型只能访问自己生成的词。有多种策略选择下一个词：
    1.  **贪心搜索 (Greedy)**：每一步选择概率最高的词。简单但次优。
    2.  **随机采样 (Random Sampling)**：根据概率分布随机抽样。能生成更多样化的文本，但可能产生低质量结果。
    3.  **集束搜索 (Beam Search)**：每一步保留 $K$ 个最可能的部分序列。是推理阶段的标准选择。$K$ 称为**束宽**。

    ```
        集束搜索 (Beam Size = 2) 示例：

        时刻 t=1:    你(0.6)  我(0.4)
                        |         |
        时刻 t=2:   你好(0.5) 你好(0.3)  你是(0.3)  我是(0.5)
                      /          \         /          \
                     /            \       /            \
        候选得分:  0.6*0.5=0.30   0.4*0.3=0.12  0.4*0.3=0.12  0.4*0.5=0.20
                       |                          |
                       v                          v
        保留Top-2:   "你好" (0.30)             "我是" (0.20)
                    继续扩展...                继续扩展...
    ```

<details>
<summary><b>💡 教师强制 vs. 计划采样 (Scheduled Sampling)</b></summary>

教师强制虽然训练稳定，但会导致 **曝光偏差 (Exposure Bias)**：训练时看到的都是正确的历史词，推理时却要面对自己生成的错误词。**计划采样**是一种折中方案：在训练过程中，以一定概率 $p$ 使用真实词（教师强制），以概率 $1-p$ 使用模型自己生成的词。随着训练进行，$p$ 逐渐降低，让模型学会修正自己的错误。
</details>

# 三、注意力机制 (Attention)

## 3.1 动机：信息瓶颈

-   **问题**：在基本的Seq2Seq模型中，编码器必须将整个输入序列的信息压缩到一个固定长度的向量 $C$ 中。对于长序列，这成为信息瓶颈，解码器在生成后期会"遗忘"输入开头的关键信息。

-   **核心思想**：不让解码器只盯着一个固定的 $C$，而是允许它在生成每个输出词时，**动态地、有选择地"回看"输入序列的不同部分**。

## 3.2 通用注意力公式 (Query-Key-Value)

1.  **编码器**：为每个输入位置 $i$ 计算一个键 (Key) $k_i$ 和一个值 (Value) $v_i$。
2.  **解码器**：在生成第 $t$ 个输出时，基于当前状态计算一个查询 (Query) $q_t$。
3.  **注意力权重**：计算 $q_t$ 与所有 $k_i$ 的**相似度**（通常用点积），并通过 Softmax 归一化为权重 $a_{ti}$。
    <br>
    $$ a_{ti} = \frac{\exp(\text{score}(q_t, k_i))}{\sum_{j} \exp(\text{score}(q_t, k_j))} $$
    <br>
4.  **上下文向量**：根据注意力权重对所有 $v_i$ 进行加权求和。
    <br>
    $$ c_t = \sum_{i} a_{ti} v_i $$
    <br>
5.  **解码更新**：将 $c_t$ 与解码器当前状态 $h_t$ 或当前输出词结合，用于生成下一个词。

    ```
        注意力机制工作流程：

        解码器状态 h_t
              |
              | (通过线性变换)
              v
              q_t (查询)
              |
              +-----------------+ ... +-----------------+
              |                 |     |                 |
              v                 v     v                 v
             点积               点积   点积              点积
              |                 |     |                 |
              v                 v     v                 v
             k_1 (键)          k_2    ...               k_N (键)
              |                 |     |                 |
              +-----------------+-----+-----------------+
                           |
                           v
                    Softmax (归一化)
                           |
                           v
                   注意力权重 a_t1...a_tN
                           |
                           v
                  加权求和 Σ a_ti * v_i
                           |
                           v
                      上下文向量 c_t
    ```

## 3.3 注意力评分函数 (Score Function)

| 函数名                | 公式                           | 参数                 |
| :-------------------- | :----------------------------- | :------------------- |
| **点积**              | $q_t^T k_i$                    | 无                   |
| **缩放点积**          | $\frac{q_t^T k_i}{\sqrt{d_k}}$ | 无 (Transformer使用) |
| **加性 (Bahdanau)**   | $v_a^T \tanh(W_a[q_t; k_i])$   | $v_a, W_a$           |
| **点积+投影 (Luong)** | $q_t^T W_a k_i$                | $W_a$                |

## 3.4 注意力机制的好处

1.  **解决信息瓶颈**：解码器可以直接访问整个输入序列，极大地提升了对长序列的处理能力。
2.  **可解释性**：可视化注意力权重 $a_{ti}$ 可以让我们看到模型在生成每个词时"看"的是输入的哪个部分。例如，在翻译中，生成"apfel"时，注意力会集中指向输入中的"apple"。
3.  **缓解梯度消失**：为误差信号提供了从输出到输入的"直通车"。

# 四、Transformer

## 4.1 核心思想："注意力就是一切"

Transformer 完全摒弃了循环神经网络，仅依赖**自注意力 (Self-Attention)** 和前馈网络。这带来了两个革命性优势：
1.  **全局感受野**：任意两个位置的词都可以直接交互，路径长度 $O(1)$，完美捕获长距离依赖。
2.  **高度可并行**：每个词的表示可以独立计算，充分利用 GPU 进行并行训练。

## 4.2 Transformer 架构详解

Transformer 是一个标准的编码器-解码器架构，但两个部分均由多个相同的"块"堆叠而成。

### 4.2.1 位置编码 (Positional Encoding)

-   **为什么需要**：自注意力机制本身是**置换不变的**（permutation invariant），即打乱输入顺序会得到同样的输出。必须注入位置信息。
-   **正弦位置编码 (Sinusoidal PE)**：原始论文使用的固定编码。
    <br>
    $$ PE_{(pos, 2i)} = \sin\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right) $$
    $$ PE_{(pos, 2i+1)} = \cos\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right) $$
    <br>
    -   $pos$ 是位置，$i$ 是维度索引。
    -   **性质**：$PE_{pos+k}$ 可以表示为 $PE_{pos}$ 的线性函数，使模型易于学习相对位置。
-   **可学习位置编码**：将位置编码作为一个可学习的嵌入矩阵，让模型自己学习。在多数现代LLM（如GPT, LLaMA）中更常见。
-   **旋转位置编码 (RoPE)**：LLaMA, Qwen 等模型使用。通过旋转矩阵来编码相对位置，是目前最主流的方法之一。

    ```
        正弦位置编码的可视化：

        维度
         ^
         |  ~~~~~~         ~~~~~~         ~~~~~~        (低频，长周期)
         |       ~~~~~~~         ~~~~~~~         ~~~
         |            ~~~~~~~~~         ~~~~~~~~~
         |                 ~~~~~~~~~~         ~~~~~~~~
         +------------------------------------------------> 位置 (pos)

       不同维度有不同的振荡频率，为每个位置提供了唯一的"指纹"。
    ```

### 4.2.2 编码器 (Encoder) - BERT 的基础

-   **输入**：词嵌入 + 位置编码。
-   **核心组件**：
    1.  **多头自注意力 (Multi-Head Self-Attention)**：
        -   **自注意**：$Q, K, V$ 都来自同一个输入序列。
        -   **多头**：将 $Q, K, V$ 线性投影多次，并行执行多次注意力计算，最后将结果拼接。每个"头"可以学习关注不同类型的关系（如语法、共指、语义）。
    2.  **前馈网络 (FFN)**：一个简单的两层 MLP，用于对每个位置独立地进行非线性变换。
        <br>
        $$ \text{FFN}(x) = \max(0, xW_1 + b_1)W_2 + b_2 $$
        <br>
    3.  **残差连接 & 层归一化 (Add & Norm)**：每个子层（注意力、FFN）后都紧跟。
        <br>
        $$ \text{Output} = \text{LayerNorm}(x + \text{Sublayer}(x)) $$
        <br>

### 4.2.3 解码器 (Decoder) - GPT 的基础

解码器结构与编码器类似，但在自注意力层之间插入了一个**交叉注意力 (Cross-Attention)** 层。

1.  **掩码自注意力 (Masked Self-Attention)**：在计算自注意力时，**屏蔽（mask）** 掉当前位置之后的信息。这保证了生成下一个词时，模型只能看到已经生成的词，符合自回归特性。
2.  **交叉注意力 (Cross-Attention)**：
    -   **查询 ($Q$)**：来自解码器上一层的输出。
    -   **键 ($K$) 和值 ($V$)**：来自**编码器**的最终输出。
    -   **作用**：让解码器在生成每个词时，可以"关注"输入序列的不同部分。

    ```
        Transformer 编码器和解码器块结构：

        +----------------------+      +----------------------+
        |     编码器块 x N      |      |     解码器块 x N      |
        +----------------------+      +----------------------+
        |                      |      |                      |
        |   +--------------+   |      |   +--------------+   |
        |   | 多头自注意力  |   |      |   | 掩码多头自注力 |   |
        |   +--------------+   |      |   +--------------+   |
        |         |            |      |         |            |
        |    +----+----+       |      |    +----+----+       |
        |    | Add & Norm|     |      |    | Add & Norm|     |
        |    +----+----+       |      |    +----+----+       |
        |         |            |      |         |            |
        |   +--------------+   |      |   +--------------+   |
        |   |   前馈网络   |   |      |   | 交叉注意力   | ◄──┐ (来自编码器)
        |   +--------------+   |      |   +--------------+   |    |
        |         |            |      |         |            |    |
        |    +----+----+       |      |    +----+----+       |    |
        |    | Add & Norm|     |      |    | Add & Norm|     |    |
        |    +----+----+       |      |    +----+----+       |    |
        |         |            |      |         |            |    |
        +---------+------------+      |   +--------------+   |    |
                                      |   |   前馈网络   |   |    |
                                      |   +--------------+   |    |
                                      |         |            |    |
                                      |    +----+----+       |    |
                                      |    | Add & Norm|     |    |
                                      |    +----+----+       |    |
                                      |         |            |    |
                                      +---------+------------+----+
                                                |
                                                v
                                            输出概率
    ```

## 4.3 Transformer 的衍生变体

| 变体                | 架构              | 预训练任务                                         | 特点                                       | 代表模型            |
| :------------------ | :---------------- | :------------------------------------------------- | :----------------------------------------- | :------------------ |
| **Encoder-Only**    | 仅编码器          | **掩码语言模型 (MLM)**：预测被 `[MASK]` 遮挡的词。 | 双向上下文理解能力强，擅长分类、抽取任务。 | **BERT**, RoBERTa   |
| **Decoder-Only**    | 仅解码器 (带掩码) | **因果语言模型 (CLM)**：标准的下一词预测。         | 生成能力强，是目前LLM的主流架构。          | **GPT 系列**, LLaMA |
| **Encoder-Decoder** | 编码器+解码器     | 降噪自编码 (如T5)                                  | 适合输入输出差异大的序列转换任务。         | **T5**, BART        |

## 4.4 高效注意力的优化

标准注意力的计算复杂度为 $O(N^2 \cdot d)$，在处理长序列时成本高昂。

-   **FlashAttention**：一种 IO-感知的精确注意力算法。
    -   **核心思想**：将注意力计算分块进行，使其在 GPU 的**高速共享内存 (SRAM)** 中完成，而非慢速的**高带宽内存 (HBM)**，从而避免了频繁的内存读写。
    -   **效果**：在保持数学等价性的同时，大幅提升速度并降低内存占用。
-   **分组查询注意力 (Grouped-Query Attention, GQA)**：多查询注意力和多头注意力的折中。将查询头分组，每组共享一对键值头。在保持性能的同时，显著降低推理时的内存带宽消耗（特别是**键值缓存 (KV-Cache)** 的读取）。LLaMA 2/3 等模型使用。
-   **KV-Cache**：在自回归生成时，对于已生成的 $t$ 个词，其键值对在计算第 $t+1$ 个词时不会改变。缓存它们可以避免重复计算，这是LLM推理加速的关键技术。

# 五、大语言模型 (LLM)

## 5.1 LLM 训练三阶段

现代 LLM 的训练通常分三步走：

1.  **预训练 (Pre-training)**：在海量、多样化的原始文本（如网页、书籍、代码）上进行下一词预测。这是吸收知识的主要阶段。
    -   **损失**：$ \mathcal{L}_{\text{pretrain}} = -\frac{1}{T} \sum_{t=1}^{T} \log P(w_t \mid w_{<t}) $
    -   **缩放法则 (Scaling Law)**：模型的性能与参数量、数据量、计算量呈**幂律关系**。这意味着盲目扩大模型不如同时扩大高质量数据。

2.  **监督微调 (Supervised Fine-Tuning, SFT)**：使用高质量的**指令-回答 (instruction-response)** 对，对预训练模型进行进一步训练。目的是教会模型遵循用户的指令。
    -   **数据特点**：数据量小得多（数万到数百万），但质量要求极高。包含问答、头脑风暴、代码生成、角色扮演等多种任务。

3.  **从人类反馈中强化学习 (RLHF)**：进一步用强化学习优化模型，使其行为更符合人类偏好（如有用、诚实、无害）。
    -   **步骤**：
        1.  **训练奖励模型 (Reward Model, RM)**：让人类对模型生成的多条回答进行排序，训练一个模型来模仿人类的偏好打分。
        2.  **用强化学习优化**：使用**近端策略优化 (PPO)** 等算法，以奖励模型的分数作为信号，微调LLM。

    ```
        RLHF 流程：

        步骤1: SFT模型          步骤2: 训练奖励模型         步骤3: PPO优化

        指令 ---> [SFT模型] ---> 回答A   指令 ---> [SFT模型] ---> 回答B  指令 ---> [LLM] ---> 回答
                     |            |                     |            |           |         |
                     |            v                     |            v           v         v
                     +---------> 人类排序 (A > B > C)    +---------> [奖励模型]   奖励 <--- 奖励模型
                                  |                                     ^           |
                                  v                                     |           v
                               [奖励模型]  <------------------------- 得分         更新参数
    ```

## 5.2 缩放法则 (Scaling Laws)

-   **发现** (Kaplan et al., OpenAI, 2020)：测试集损失 $L$ 与参数量 $N$、数据量 $D$、计算量 $C$ 呈幂律关系。
    <br>
    $$ L(N, D) = E + \frac{A}{N^\alpha} + \frac{B}{D^\beta} $$
    <br>
    -   $E$ 是数据本身的熵（不可约损失），$A, B > 0$，$0 < \alpha, \beta < 1$。
-   **Chinchilla 法则** (Hoffmann et al., DeepMind, 2022)：修正了 OpenAI 的结论，认为**模型大小 $N$ 和训练数据 $D$ 应该等比例缩放**。对于 $C$ 计算预算，最优的分配是 $N \propto C^{0.5}, D \propto C^{0.5}$，即参数和数据量应保持平衡。
-   **实践趋势**：目前的 LLM 普遍"过度训练"（overtrained），即数据量远大于 Chinchilla 法则的建议（如 Llama 3 用了 15T tokens 训练 70B 模型）。这是因为推理成本远高于训练成本，人们更愿意训练一个稍大的模型，以减少推理时的幻觉和提升能力。

<details>
<summary><b>📊 Scaling Laws 实战应用</b></summary>

有了缩放法则，你可以在用全部算力训练大模型之前：
1.  在小规模（如参数量 10M-100M）上进行大量实验，调整超参数。
2.  根据小规模实验的结果，**精确预测**在目标大规模（如 7B）上训练 $X$ 个 token 后的损失。
3.  这避免了"押注"式的大规模训练，大大降低了试错成本。

DeepMind 正是利用 Chinchilla 法则，预测并训练出了一个 70B 的模型，其性能优于当时 178B 的 Jurassic-1 模型。
</details>

## 5.3 推理能力提升：思维链 (Chain-of-Thought, CoT)

-   **思想**：让模型在给出最终答案前，先生成中间的推理步骤。
-   **方法**：在提示中加入 "Let's think step by step." 或在 SFT 数据中加入结构化的推理过程。
-   **效果**：显著提升模型在数学、逻辑推理、代码生成等复杂任务上的表现。
-   **扩展**：
    -   **自洽性 (Self-Consistency)**：对同一个问题，让模型生成多条 CoT 路径，最终投票选出最一致的答案。
    -   **思维树 (Tree-of-Thoughts, ToT)**：在 CoT 的基础上，探索和回溯不同的推理分支。

    ```
        思维链 (CoT) 示例：

        问：罗杰有5个网球。他买了2罐网球，每罐有3个。他现在有多少个？

        不加 CoT (可能犯错)：
        答：11个。

        加 CoT (正确)：
        答：罗杰一开始有5个球。2罐网球，每罐3个，一共是 2*3=6 个球。
           5 + 6 = 11。
           答案是 11。
    ```

## 5.4 LLM 的评估

评估 LLM 的"好坏"非常困难，需要多维度进行。

| 评估维度       | 代表数据集/方法        | 评估内容                            |
| :------------- | :--------------------- | :---------------------------------- |
| **知识能力**   | MMLU, Hellaswag        | 多学科问答、常识推理                |
| **推理能力**   | GSM8K, MATH, HumanEval | 数学解题、代码生成                  |
| **指令遵循**   | IFEval, MT-Bench       | 模型能否精确遵守指令的格式/内容要求 |
| **对齐与安全** | BBQ, ToxiGen           | 偏见、有害内容、幻觉                |
| **人类偏好**   | Chatbot Arena          | 匿名投票，直接比较模型生成质量      |

-   **关键挑战**：
    -   **数据污染**：评测集可能不小心混入了训练数据中，导致分数虚高。
    -   **涌现能力**：某些能力在小模型上完全不存在，在模型规模超过某个阈值后突然出现，这使得在小规模上预测大模型行为变得困难。

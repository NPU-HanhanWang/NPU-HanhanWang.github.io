# 一、从有限响应到无限响应：RNN的动机

## 1.1 序列建模的两种范式

| 模型类型         | 数学形式                                    | 特点                                             | 代表网络                     |
| :--------------- | :------------------------------------------ | :----------------------------------------------- | :--------------------------- |
| **有限响应模型** | $Y_t = f(X_t, X_{t-1}, \dots, X_{t-N})$     | 记忆长度固定为 $N$，增加记忆需增加参数           | 时延神经网络 (TDNN) / 1D CNN |
| **无限响应模型** | $Y_t = f(X_t, X_{t-1}, \dots, X_{-\infty})$ | 理论上有无限记忆，一个输入可影响无限远的未来输出 | **循环神经网络 (RNN)**       |

-   **关键洞察**：CNN/TDNN 是"有限冲激响应"系统，而 RNN 是"无限冲激响应"系统。今天的输入可以永远影响未来的输出。

## 1.2 RNN 的状态空间形式

最通用的 RNN 形式是**状态空间模型 (State-Space Model)**：
<br>
$$ h_t = f(x_t, h_{t-1}) $$
$$ y_t = g(h_t) $$
<br>
-   $h_t$：**隐藏状态 (Hidden State)**，是对过去所有信息的一个**压缩摘要**。
-   $f(\cdot)$：状态转移函数（通常是一个带激活函数的全连接层）。
-   $g(\cdot)$：输出函数（可以是全连接层 + Softmax）。

    ```
        标准 RNN 单元展开图 (unrolled over time)：

        时刻:   t-1         t           t+1
        输入:   x_{t-1} ---> x_t ---> x_{t+1}
                 |           |           |
                 v           v           v
        状态:   h_{t-1} ---> h_t ---> h_{t+1}
                 |           |           |
                 v           v           v
        输出:   y_{t-1}      y_t         y_{t+1}

        关键：所有时刻的权重矩阵 W_xh, W_hh, W_hy 是**共享**的。
    ```

<details>
<summary><b>📝 RNN 为什么能比 MLP 更高效地解决某些问题？</b></summary>

以奇偶性校验 (Parity) 问题为例：判断一个任意长度的二进制序列中1的个数是奇数还是偶数。

-   **MLP 做法**：输入长度固定为 $N$，需要学习 $2^N$ 种输入模式，需要海量数据和超大网络，且无法泛化到其他长度。
-   **RNN 做法**：只需一个很小的循环网络，在内部状态 $h_t$ 中维护一个"累积和 mod 2"的计数器。网络通过 $h_t = \text{mod}(h_{t-1} + x_t, 2)$ 这样的简单逻辑即可工作，参数量恒定，且能泛化到任意长度的输入。

RNN 通过**时间上的参数共享**和**状态压缩**，将序列处理问题从"学习所有可能的模式"转化为"学习状态的更新规则"，极大地提高了样本效率。
</details>

# 二、训练RNN：随时间反向传播 (BPTT)

## 2.1 核心思想

将展开的 RNN 视为一个**层数等于序列长度的、权值共享的超深前馈网络**，然后应用标准的反向传播算法。

-   **共享参数处理**：所有权重 $W_{xh}, W_{hh}, W_{hy}$ 在每个时间步是相同的。因此，最终的梯度是所有时间步梯度的**和**。
    <br>
    $$ \frac{\partial \text{Loss}}{\partial W_{hh}} = \sum_{t=1}^{T} \frac{\partial \text{Loss}_t}{\partial W_{hh}} $$
    <br>

## 2.2 BPTT 伪代码

假设只有一个隐藏层。

```
    # 前向传播 (存储所有中间值)
    h_{-1} = h_init  # 初始状态，通常为零向量或可学习参数
    for t = 0 to T-1:
        z_h[t] = W_xh * x[t] + W_hh * h_{t-1} + b_h
        h[t] = tanh(z_h[t])
        z_y[t] = W_hy * h[t] + b_y
        y[t] = softmax(z_y[t])
        Loss += divergence(y[t], d[t])

    # 反向传播 (BPTT)
    dL_dh_next = 0  # 从未来传回的梯度
    for t = T-1 down to 0:
        # 1. 输出层的梯度
        dL_dz_y[t] = dL_dy[t] * softmax_derivative(y[t])
        dL_dW_hy += outer(h[t], dL_dz_y[t])
        dL_db_y += dL_dz_y[t]
        dL_dh[t] = W_hy^T * dL_dz_y[t] + dL_dh_next  # 关键：加上从下一时刻传回的梯度

        # 2. 隐藏层的梯度 (通过 tanh)
        dL_dz_h[t] = dL_dh[t] * tanh_derivative(h[t])

        # 3. 权重梯度 (累积)
        dL_dW_xh += outer(x[t], dL_dz_h[t])
        dL_dW_hh += outer(h[t-1], dL_dz_h[t])
        dL_db_h += dL_dz_h[t]

        # 4. 梯度向更早时间步传播
        dL_dh_next = W_hh^T * dL_dz_h[t]

    # 用累积的梯度进行参数更新
```

# 三、RNN 的稳定性分析与梯度问题

## 3.1 BIBO 稳定性

-   **定义**：对于有界输入，系统是否产生有界输出。
-   **线性递归分析**：考虑一个简化的线性 RNN，$h_t = W_{hh} h_{t-1} + x_t$。通过递推：
    <br>
    $$ h_t = W_{hh}^t h_0 + \sum_{k=0}^{t-1} W_{hh}^k x_{t-k} $$
    <br>
-   **关键结论**：长期行为由 $W_{hh}$ 的最大特征值 $\lambda_{\text{max}}$ 主导。
    -   若 $|\lambda_{\text{max}}| > 1$：长期记忆**爆炸 (Explode)**，数值不稳定。
    -   若 $|\lambda_{\text{max}}| < 1$：长期记忆**消失 (Vanish)**，无法记住太久远的信息。
    -   若 $|\lambda_{\text{max}}| = 1$：理论上可以长期保持记忆（如酉矩阵）。

## 3.2 梯度消失与爆炸

这是训练深度网络（包括深层MLP和RNN）的根本性问题。

-   **原因**：梯度在反向传播时，需要通过多个权重矩阵和激活函数导数的连乘。
-   **连乘效应**：
    <br>
    $$ \frac{\partial \text{Loss}}{\partial h_{k}} = \frac{\partial \text{Loss}}{\partial h_{T}} \prod_{t=k+1}^{T} \left( \frac{\partial h_t}{\partial h_{t-1}} \right) = \frac{\partial \text{Loss}}{\partial h_{T}} \prod_{t=k+1}^{T} \left( W_{hh}^T \cdot \text{diag}(f'(z_{t-1})) \right) $$
    <br>
-   **结论**：
    -   $\|W_{hh}\|$ 和 $\|f'(\cdot)\|$（通常 < 1）的反复相乘，会使梯度呈指数级缩小（**消失**）。
    -   这导致网络无法学习到长时间跨度的依赖关系（例如，在长文本中，一个单词对远在20步之前的另一个单词的依赖）。

-   **现实影响**：标准RNN在实践中几乎无法记住超过5-10步的信息。

    ```
        梯度消失的直观图示：

        损失 L_t
           |
           v
        (∂L/∂h_t)   <-- 这里梯度还很大
           |
           v
        (∂L/∂h_{t-1}) * W_hh * f'(...)
           |
           v   (乘以小于1的数)
           v
        (∂L/∂h_{t-2})  <-- 梯度开始变小
           |
           ... (反复乘以小于1的数)
           |
           v
        (∂L/∂h_{t-10}) <-- 梯度几乎为0，无法更新早期权重
    ```

# 四、长短期记忆网络 (LSTM)

## 4.1 核心思想：门控恒误差轮 (CEC)

LSTM通过引入一个**独立的、线性的"细胞状态" (Cell State, $C_t$)** 和**门控机制**来解决梯度消失问题。关键想法是让信息在细胞状态上流动时，只经历**线性变换**，从而让梯度可以无衰减地传播。

## 4.2 LSTM 单元内部结构

一个LSTM单元有3个门控信号和一个记忆细胞：

1.  **遗忘门 (Forget Gate, $f_t$)**：决定从过去的细胞状态 $C_{t-1}$ 中丢弃多少信息。
2.  **输入门 (Input Gate, $i_t$)**：决定是否将新的候选信息 $\tilde{C}_t$ 加入到细胞状态中。
3.  **输出门 (Output Gate, $o_t$)**：决定从当前的细胞状态 $C_t$ 中输出多少信息到隐藏状态 $h_t$。

    ```
                    ┌─────────────────────────────────────┐
                    │              LSTM Cell              │
                    │                                     │
        h_{t-1} ────┼─┬─────────────────────────┬─────────┼───► h_t
                    │ │                         │         │
                    │ │    ┌────┐     ┌────┐    │         │
        x_t ────────┼─┼───►│ σ  │────►│    │    │         │
                    │ │    └────┘     └─┬──┘    │         │
                    │ │                 │       │         │
                    │ │    ┌────┐       │       │         │
                    │ ├───►│ σ  │───────┼──────►│    ┌────┤
                    │ │    └────┘       │       │    │    │
                    │ │                 │       │    │    │
                    │ │    ┌────┐       │       │    │    │
                    │ └───►│tanh│───────┼──────►│    │    │

                                     └────┘     │         │
    ```

### 4.2.1 LSTM 前向传播公式

设 $x_t$ 为当前输入，$h_{t-1}$ 为上一时刻隐藏状态，$C_{t-1}$ 为上一时刻细胞状态。

1.  **遗忘门**：
    <br>
    $$ f_t = \sigma(W_f \cdot [h_{t-1}, x_t] + b_f) $$
    <br>
2.  **输入门**与**候选细胞状态**：
    <br>
    $$ i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i) $$
    $$ \tilde{C}_t = \tanh(W_C \cdot [h_{t-1}, x_t] + b_C) $$
    <br>
3.  **细胞状态更新**（核心线性路径）：
    <br>
    $$ C_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_t $$
    <br>
4.  **输出门**与**隐藏状态更新**：
    <br>
    $$ o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o) $$
    $$ h_t = o_t \odot \tanh(C_t) $$
    <br>

-   $\odot$ 表示逐元素乘法 (Hadamard product)。
-   $\sigma$ 是 Sigmoid 函数，输出在 (0,1) 之间，作为"门"的开度。

## 4.3 为什么LSTM能缓解梯度消失？

-   **梯度高速公路**：从 $C_{t-1}$ 到 $C_t$ 的更新是**线性**的：$\frac{\partial C_t}{\partial C_{t-1}} = f_t$。
-   **遗忘门的调节**：$f_t$ 是一个由网络学习的、介于0和1之间的值。
    -   当网络需要长期记忆时，它会设置 $f_t \approx 1$，此时 $\frac{\partial C_t}{\partial C_{t-1}} \approx 1$，梯度可以几乎无损地传播。
    -   当网络需要遗忘旧信息时，它会设置 $f_t \approx 0$。
-   **与标准RNN的对比**：标准RNN的梯度回传需要通过 $\tanh'(z) \cdot W_{hh}$，这个值几乎总是小于1，导致梯度必会消失。LSTM则有机会"打开遗忘门"，让梯度畅通无阻。

## 4.4 门控循环单元 (GRU)：LSTM的简化版

GRU将遗忘门和输入门合并为一个**更新门 (Update Gate, $z_t$)**，并将细胞状态和隐藏状态合并。

-   **公式**：
    <br>
    $$ z_t = \sigma(W_z \cdot [h_{t-1}, x_t]) $$
    $$ r_t = \sigma(W_r \cdot [h_{t-1}, x_t]) $$
    $$ \tilde{h}_t = \tanh(W \cdot [r_t \odot h_{t-1}, x_t]) $$
    $$ h_t = (1 - z_t) \odot h_{t-1} + z_t \odot \tilde{h}_t $$
    <br>
-   **优点**：参数更少，计算效率更高，在中小规模数据集上表现与LSTM相当。

# 五、序列到序列模型与CTC

## 5.1 问题分类

| 类型           | 输入     | 输出     | 对齐方式             | 典型任务               | 模型                      |
| :------------- | :------- | :------- | :------------------- | :--------------------- | :------------------------ |
| **多对一**     | 序列     | 单个向量 | 只在最后一步计算损失 | 情感分类、语音指令识别 | RNN + 最后输出            |
| **同步多对多** | 序列     | 等长序列 | 每一步都有对应标签   | 词性标注 (POS Tagging) | 双向RNN + 逐帧Softmax     |
| **异步多对多** | 序列     | 变长序列 | 对齐未知             | 语音识别、机器翻译     | Seq2Seq + Attention / CTC |
| **一对多**     | 单个向量 | 序列     | 从单点开始生成       | 图像描述生成           | Encoder-Decoder           |

## 5.2 训练异步序列模型：对齐问题

在语音识别中，我们有一段音频 $X_1...X_T$ 和对应的文本（音素序列）$S_1...S_K$，但不知道每个音素在音频中出现的**确切起止时间**。这被称为**未对齐数据**。

-   **对齐 (Alignment)**：将长度为 $K$ 的压缩序列"扩展"为长度为 $T$ 的时间同步序列。一个对齐示例：`/B/ /B/ /B/ - /IY/ /IY/ /F/ - /IY/`（`-` 表示静音或边界）。
-   **"压缩" (Compression)**：将时间同步序列中相邻的重复符号压缩为一个，并删除`-`，即可还原回原序列。

## 5.3 解决方案1：Viterbi 训练 (硬对齐)

-   **思想**：在每一步迭代中，先找出当前模型下**最可能的**对齐路径，然后用这个对齐作为"真实标签"去训练模型（类似EM算法的"E步"）。
-   **Viterbi 算法**：一种动态规划算法，用于在如图所示的格子图中找到一条得分最高的路径。路径的得分是路径上所有节点概率的乘积。
-   **缺点**：容易陷入局部最优。如果初始对齐是错误的，模型会强化这个错误。

## 5.4 解决方案2：CTC 训练 (软对齐)

-   **思想**：不对齐，而是**对所有可能的对齐路径进行平均**。
    <br>
    $$ \text{Loss} = - \sum_{t} \mathbb{E}_{\text{Alignments}}[\log Y_t(s_t)] $$
    <br>
-   **前向-后向算法 (Forward-Backward Algorithm)**：一种高效计算所有路径总概率的DP算法。
    -   $\alpha(t, r)$：到达节点 $(t, r)$（时刻 $t$ 对齐到第 $r$ 个音素）的所有**前缀**路径的概率和。
    -   $\beta(t, r)$：从节点 $(t, r)$ 出发到终点的所有**后缀**路径的概率和。
    -   节点 $(t, r)$ 的总概率为 $\alpha(t, r) \cdot \beta(t, r)$。
    -   后验概率 $\gamma(t, r)$：给定输入 $X$ 和目标序列 $S$，时刻 $t$ 对齐到第 $r$ 个音素的概率。
    <br>
    $$ \gamma(t, r) = \frac{\alpha(t, r) \beta(t, r)}{\sum_{r'} \alpha(t, r') \beta(t, r')} $$
    <br>
-   **CTC 损失**：
    <br>
    $$ \text{Loss}_{\text{CTC}} = -\sum_{t} \sum_{r} \gamma(t, r) \log y_t^{S(r)} $$
    <br>
    其梯度为：
    <br>
    $$ \frac{\partial \text{Loss}_{\text{CTC}}}{\partial y_t^l} = - \sum_{r: S(r)=l} \frac{\gamma(t, r)}{y_t^l} $$
    <br>

    ```
        CTC 前向-后向算法图示：

        目标音素序列 S = /B/, /IY/, /F/, /IY/
        扩展后（含blank）S' = -, B, -, IY, -, F, -, IY, -
        行索引 r =  0   1   2   3   4   5   6   7   8

        时间 t → (T帧)
                +---+---+---+---+---+
        r=0 (-): |α0|   |   |   |β0|
                +---+---+---+---+---+
        r=1 (B): |   |α1|   |   |β1|
                +---+---+---+---+---+
        r=2 (-): |   |   |α2|   |β2|
                +---+---+---+---+---+
         ...     ...   ...   ...   ...
                +---+---+---+---+---+
        r=8 (-): |   |   |   |   |β8|
                +---+---+---+---+---+

        α(t,r) 从左上向右下计算
        β(t,r) 从右下向左上计算
        节点的后验概率 ∝ α(t,r) * β(t,r)
    ```

## 5.5 处理重复符号：空白标签 (Blank)

-   **问题**：对于 "BALL" 这样的词，"L" 出现了两次。一个简单的"重复压缩"规则无法区分 "BAL"（一个L）和 "BALL"（两个L）。
-   **解决方案**：在目标符号集中引入一个特殊的 **"空白" (Blank, 记为 `-`)** 标签。
    -   **规则**：每个真实符号可以被重复多次（表示它持续了多帧）。但是，**相邻的两个相同符号之间，必须至少有一个空白标签**。
    -   **编码示例**：
        -   "BALL" $\rightarrow$ `B - A - L - L` 或 `B - A - L L` 等。
        -   "BAL" $\rightarrow$ `B - A - L`。
-   **效果**：这使得对齐和压缩过程变得无歧义。

## 5.6 CTC 解码

-   **贪心解码 (Greedy Decoding)**：在每个时刻直接选取概率最高的符号，然后压缩（移除空白、合并重复）。简单快速，但次优。
-   **集束搜索 (Beam Search)**：维护多个候选路径，在每个时刻进行扩展和剪枝。通常在实现时，会为"以空白结尾"和"以非空白结尾"的路径维护两个不同的分数列表，以正确处理重复符号的规则。

# 一、为什么需要强化学习？

## 1.1 监督学习的局限

-   **监督学习**：从**静态、独立同分布 (i.i.d)** 的标注数据集中学习输入到输出的映射。
-   **局限**：无法处理需要**序列决策 (Sequential Decision Making)** 的问题。例如，下棋、机器人行走、自动驾驶等。

## 1.2 强化学习的核心范式

强化学习 (RL) 通过**智能体 (Agent)** 与**环境 (Environment)** 的**试错 (Trial-and-Error)** 交互来学习最优策略。

    ```
        强化学习交互循环：

                           +-------------------+
                           |                   |
                           |     环境 (Env)     |
                           |                   |
                           +-------------------+
                                    ^
                                    | 状态 S_t, 奖励 R_t
                                    |
        +--------------------------+ |
        | 智能体 (Agent)            | |
        |                          | |
        |  策略 π: S -> A          | |
        |                          | |
        +--------------------------+ |
                                    |
                    动作 A_t ---------+
    ```

-   **智能体 (Agent)**：学习者和决策者。
-   **环境 (Environment)**：智能体与之交互的外部世界。
-   **状态 (State, $S_t$)**：环境在时间 $t$ 的表示。
-   **动作 (Action, $A_t$)**：智能体在状态 $S_t$ 下采取的行为。
-   **奖励 (Reward, $R_t$)**：环境对动作 $A_t$ 的即时反馈信号。RL 的目标是最大化**累积奖励**，而不仅仅是即时奖励。

# 二、马尔可夫决策过程 (MDP)

## 2.1 MDP 的定义

几乎所有 RL 问题都可以形式化为马尔可夫决策过程 (Markov Decision Process, MDP)。MDP 由以下元组定义：$(S, A, P, R, \gamma)$。

-   $S$：有限的状态集合。
-   $A$：有限的动作集合。
-   $P$：状态转移概率，$P(s' \mid s, a)$ 是在状态 $s$ 采取动作 $a$ 后转移到 $s'$ 的概率。
-   $R$：奖励函数，$R(s, a, s')$ 是在状态 $s$ 采取动作 $a$ 转移到 $s'$ 后获得的即时奖励。
-   $\gamma$：折扣因子 (Discount Factor, $0 \le \gamma \le 1$)，用于平衡即时奖励和未来奖励的重要性。

## 2.2 策略 (Policy) 与价值函数 (Value Function)

-   **策略 (Policy, $\pi$)**：定义了智能体的行为方式。可以是确定性的 $a = \pi(s)$ 或随机性的 $\pi(a \mid s) = P(A_t = a \mid S_t = s)$。
-   **状态价值函数 (State-Value Function)**：从状态 $s$ 开始，遵循策略 $\pi$ 所能获得的期望累积折扣奖励。
    <br>
    $$ V_{\pi}(s) = \mathbb{E}_{\pi} \left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t = s \right] $$
    <br>
-   **动作价值函数 (Action-Value Function)**：在状态 $s$ 采取动作 $a$，然后遵循策略 $\pi$ 所能获得的期望累积折扣奖励。
    <br>
    $$ Q_{\pi}(s, a) = \mathbb{E}_{\pi} \left[ \sum_{k=0}^{\infty} \gamma^k R_{t+k+1} \mid S_t = s, A_t = a \right] $$
    <br>

## 2.3 贝尔曼方程 (Bellman Equation) - 最优性的核心

贝尔曼方程是 RL 中几乎所有算法的基石。它揭示了当前状态的价值与未来状态价值之间的递归关系。

-   **贝尔曼期望方程 (对于策略 $\pi$)**：
    <br>
    $$ V_{\pi}(s) = \sum_{a} \pi(a \mid s) \sum_{s', r} P(s', r \mid s, a) \left[ r + \gamma V_{\pi}(s') \right] $$
    <br>
-   **贝尔曼最优方程 (对于最优策略 $\pi^*$)**：
    <br>
    $$ V_{*}(s) = \max_{a} \sum_{s', r} P(s', r \mid s, a) \left[ r + \gamma V_{*}(s') \right] $$
    <br>
    $$ Q_{*}(s, a) = \sum_{s', r} P(s', r \mid s, a) \left[ r + \gamma \max_{a'} Q_{*}(s', a') \right] $$
    <br>

<details>
<summary><b>💡 贝尔曼方程的直觉</b></summary>

贝尔曼方程表达的是一种"自洽"性：一个状态的最优价值，等于你从这个状态出发，采取最优动作后，**立刻获得的奖励加上你进入下一个状态后的最优价值**。这就像一句古话："千里之行，始于足下"，只不过"足下"这一步的决策要考虑到"千里"之外的总和。
</details>

# 三、动态规划 (Dynamic Programming)

## 3.1 核心思想

当环境的**模型**（即状态转移概率 $P$ 和奖励函数 $R$）完全已知时，我们可以使用动态规划来求解 MDP。

-   **策略评估 (Policy Evaluation)**：迭代地更新价值函数，使其收敛到当前策略的真实价值 $V_{\pi}$。
    <br>
    $$ V_{k+1}(s) = \sum_{a} \pi(a \mid s) \sum_{s', r} P(s', r \mid s, a) [r + \gamma V_k(s')] $$
    <br>
-   **策略改进 (Policy Improvement)**：根据当前的价值函数，贪心地更新策略，选择能最大化价值的动作。
    <br>
    $$ \pi'(s) = \arg\max_{a} \sum_{s', r} P(s', r \mid s, a) [r + \gamma V_{\pi}(s')] $$
    <br>
-   **策略迭代 (Policy Iteration)**：交替进行策略评估和策略改进，直到策略收敛到最优。
-   **价值迭代 (Value Iteration)**：将策略评估和策略改进合并到一步更新中。
    <br>
    $$ V_{k+1}(s) = \max_{a} \sum_{s', r} P(s', r \mid s, a) [r + \gamma V_k(s')] $$
    <br>

## 3.2 DP 的局限性

-   **需要完美模型**：在大多数实际问题中，我们并不知道 $P(s' \mid s, a)$。
-   **计算复杂度过高**：需要多次遍历整个状态空间，对于高维、连续的状态空间是**维度灾难** (Curse of Dimensionality)。

# 四、无模型强化学习 (Model-Free RL)

当环境模型未知时，智能体必须通过与环境的直接交互来学习。

## 4.1 蒙特卡洛方法 (Monte Carlo, MC)

-   **核心思想**：学习完整的回报 (Return) $G_t$，而非期望值。
    -   **$G_t$**：从一个状态开始直到回合 (Episode) 结束所获得的实际累积折扣奖励。
-   **更新规则 (MC 策略评估)**：
    <br>
    $$ V(s_t) \leftarrow V(s_t) + \alpha \left[ G_t - V(s_t) \right] $$
    <br>
-   **优点**：无偏估计（如果 $G_t$ 是真实回报）。
-   **缺点**：
    -   高方差（$G_t$ 充满了随机性）。
    -   只能用于**有终止的回合制任务 (Episodic Tasks)**。
    -   学习效率低，必须等到一个回合结束才能更新。

## 4.2 时序差分学习 (Temporal Difference, TD)

-   **核心思想**：结合了 MC 的采样和 DP 的**自举 (Bootstrapping)**。用当前的估计值来更新另一个估计值。
-   **更新规则 (TD(0) 策略评估)**：
    <br>
    $$ V(s_t) \leftarrow V(s_t) + \alpha \left[ r_{t+1} + \gamma V(s_{t+1}) - V(s_t) \right] $$
    <br>
    其中 $\delta_t = r_{t+1} + \gamma V(s_{t+1}) - V(s_t)$ 被称为 **TD 误差 (TD Error)**。

-   **优点**：
    -   低方差。
    -   **在线学习**：每步都可以更新，无需等待回合结束。
    -   适用于**非回合制任务 (Continuing Tasks)**。
    -   通常比 MC 收敛更快。

    ```
        MC vs. TD 对比：

        回合:  S1 -> S2 -> S3 -> S4 -> 结束

        MC 更新 V(S3):
        需要等到回合结束，知道从 S3 到结束的真实总奖励 G3 = r4 + r5

        TD 更新 V(S3):
        立刻用下一个状态的估计值 r4 + γ * V(S4) 来更新 V(S3)
    ```

## 4.3 从价值函数到动作价值函数 (Q-Learning)

为了找到最优策略，我们不仅需要知道状态"好不好"，更需要知道在状态下**做哪个动作"最好"**。因此，我们直接学习**动作价值函数 $Q(s, a)$**。

-   **SARSA (On-Policy)**：
    -   **更新规则**：$Q(s, a) \leftarrow Q(s, a) + \alpha [r + \gamma Q(s', a') - Q(s, a)]$。
    -   **特点**：它学习的是**当前行为策略** $\pi$ 的 $Q$ 值。下一个动作 $a'$ 是由当前策略决定的。
    -   **"On-Policy"**：用于收集数据的策略和正在被评估/改进的策略是同一个。

-   **Q-Learning (Off-Policy)**：
    -   **更新规则**：
        <br>
        $$ Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right] $$
        <br>
    -   **特点**：它学习的是**最优策略** $\pi^*$ 的 $Q$ 值。它直接从另一个策略（通常是贪心策略）的最大化结果中进行更新，无论当前行为策略是什么。
    -   **"Off-Policy"**：用于收集数据的策略（通常是 $\epsilon$-greedy 探索策略）和被学习的策略（贪心策略）是不同的。

# 五、深度强化学习 (Deep RL)

## 5.1 引入深度神经网络 (DQN)

当状态空间巨大且连续时，无法再用表格存储 $Q$ 值。解决方案是用一个**深度神经网络 (Deep Q-Network, DQN)** 来近似 $Q$ 函数：$Q(s, a; \theta) \approx Q^*(s, a)$。

-   **输入**：状态 $s$（可以是原始像素）。
-   **输出**：每个动作 $a$ 的 $Q$ 值 $Q(s, a; \theta)$。

## 5.2 训练 DQN 的三大挑战与解决方案

直接将 Q-Learning 与非线性神经网络结合会导致训练不稳定甚至发散。DQN 论文 (Mnih et al., 2015) 提出了两个关键技巧来解决此问题。

| 挑战                                    | 原因                                                                                                                       | DQN 解决方案                                                                                                                                                                             |
| :-------------------------------------- | :------------------------------------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **数据相关性 (Correlated Data)**        | 连续采样的 $(s, a, r, s')$ 高度相关，破坏了神经网络 i.i.d 的假设。                                                         | **经验回放 (Experience Replay)**：将所有经历 $(s, a, r, s')$ 存储到一个**回放缓冲区 (Replay Buffer)** 中。训练时，从中随机采样一个小批量数据，打破数据相关性。                           |
| **非平稳目标 (Non-stationary Targets)** | $Q$-Learning 的更新目标 $r + \gamma \max_{a'} Q(s', a')$ 本身依赖于正在训练的网络参数 $\theta$，导致"追逐一个移动的目标"。 | **目标网络 (Target Network)**：使用一个独立的、结构相同的**目标网络** $Q(s, a; \theta^-)$ 来计算 TD 目标。目标网络的参数 $\theta^-$ 每隔固定步数才从主网络 $\theta$ 复制一次，保持不变。 |

    ```
        DQN 架构与训练流程：

        1. 与环境交互收集经验:
           状态 s -----> [主网络 Q_θ] ---> 动作 a (ε-greedy)
               <---(s, a, r, s')---> [回放缓冲区]

        2. 从回放缓冲区随机采样小批量:
            ┌──────────────────────────────┐
            │ (s, a, r, s')  (s, a, r, s') ... │
            └──────────────────────────────┘
                           |
                           v
        3. 计算 TD 目标:
           主网络 Q_θ(s, a)              目标网络 Q_θ⁻(s', a')
                |                               |
                v                               v
           预测值 Q(s,a)               目标值 r + γ * max_{a'} Q_θ⁻(s', a')
                           |               ^
                           |_______________| (θ⁻ 固定，定期从 θ 复制)
                           |
                           v
        4. 计算损失并更新主网络:
           Loss = MSE(目标值, 预测值)
           θ = θ - η * ∇_θ Loss
    ```

# 六、从人类反馈中强化学习 (RLHF)

## 6.1 问题的由来

在现实世界的许多任务中（如聊天机器人、内容生成），定义一个明确的、可计算的奖励函数是极其困难的。什么是一个"有用"的回答？什么是一个"有趣"的故事？

## 6.2 RLHF 的核心思想

让**人类**来提供奖励信号。

1.  **收集偏好数据**：向人类展示模型生成的多个回答，请他们对这些回答进行**排序或评分**。

2.  **训练奖励模型 (Reward Model)**：训练一个神经网络 $R_{\phi}(s, a)$，它的任务是模仿人类的偏好，预测给定回答的"好"的程度。

    ```
        奖励模型训练：

        提示: "解释什么是黑洞"
        -> 模型A回答: "一个密度很大的天体..."  (人类评分: 4.5/5)
        -> 模型B回答: "我不知道..."       (人类评分: 1/5)

        损失函数: 最大化偏好对之间的差距，例如使用 Bradley-Terry 模型：
        Loss = -log( σ( R(回答A) - R(回答B) ) )
        (如果人类更喜欢 A)
    ```

3.  **用强化学习优化策略**：将训练好的奖励模型 $R_{\phi}$ 作为环境奖励的替代。使用近端策略优化 (PPO) 等 RL 算法来优化语言模型的策略 $\pi_{\theta}$，使其生成的回答能获得尽可能高的奖励分数。

    ```
        RLHF 流程：

        步骤1: 监督微调 (SFT)         步骤2: 训练奖励模型 (RM)       步骤3: 用 PPO 优化
        (可选，预热模型)

        提示 ---> [LLM] ---> 回答      (s, a) ---> [RM] ---> 奖励    提示 ---> [LLM_π] ---> 回答 ---> [RM] ---> 奖励
                  |                     ^                              |                     ^           |
                  v                     |                              v                     |           v
              监督学习                人类偏好排序                       PPO 更新 <------------ 奖励信号 ---+
    ```

# 七、RL 的挑战与未来方向

| 挑战               | 描述                                                                                                         | 研究方向                                                                                   |
| :----------------- | :----------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------- |
| **样本效率低**     | 需要与环境交互数百万甚至数十亿步才能学会一个任务，而人类只需要很少的样本。                                   | 基于模型的 RL (Model-Based RL)、元学习 (Meta-Learning)、高效探索 (Efficient Exploration)。 |
| **奖励函数设计难** | "辛辛苦苦设计的奖励函数，最后发现模型钻了空子，学到了我们不想要的行为。" (Specification Gaming)              | 逆强化学习 (Inverse RL)、RLHF、从演示中学习。                                              |
| **探索 vs. 利用**  | 如何平衡"选择已知的好动作"和"尝试可能带来更大收益的新动作"？尤其是在奖励稀疏的环境中。                       | 内在奖励 (Intrinsic Motivation)、好奇心驱动探索 (Curiosity-driven Exploration)。           |
| **稳定性和泛化性** | DRL 训练极其脆弱，超参数敏感。在训练环境中表现优异的模型，可能在测试环境（如光照、物理参数变化）中完全失效。 | 鲁棒 RL (Robust RL)、域随机化 (Domain Randomization)、泛化 RL。                            |

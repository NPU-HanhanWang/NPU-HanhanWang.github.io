### 第1步：本能直觉反应（制造共鸣 + 指出坑）

一看到**「找两个数加起来等于目标值」**，正常人第一反应绝对是**「双重循环暴力穷举呗，把每两个数都加一遍看看！」**

但这是个坑！因为——**这玩意儿就像你在一千人里面找两个人，非要把每两个人都配对握手一次，手都握断了才找到！** 数组长度只要稍微长一点（比如一万个数），电脑就要算上亿次加法，直接卡成PPT。题目最后那个「进阶」提示就在疯狂暗示：**别这么干！**

---

### 第2步：核心武器（类比迁移）

这道题真正的大招是**哈希表（unordered_map）**！

跟哪道经典题很像？跟**「点名册查人」**一模一样！想象一下你手上有本花名册（哈希表），上面记录了**「每个数字在第几页（下标）」**。

**为什么哈希表正好能解决这个痛点？**
因为暴力法的痛点在于——每看一个数，都要回头把之前所有数再翻一遍找搭档。而哈希表的绝活是：**记性极好**。我每看一个新数，只需要问哈希表一句话：**「我的搭档（target - 当前数）我之前见过没？」** 哈希表能在瞬间（O(1)时间）回答你。这就像你一边点名一边把点过的人记在脑子里，下一个同学一上台，你立马能想起来「哎，刚才是不是有个叫XXX的跟你搭伙正好？」

---

### 第3步：暴力解法演示（先感性后理性）

**先给思路**：把所有数两两配对，挨个加一遍，直到找到和等于target的那一对。

**再跑例子**：`nums = [2, 7, 11, 15], target = 9`

- 第1轮：拿 `2` 分别加 `7`（=9 ✓ 找到了！返回[0,1]）
- （如果没找到就继续）拿 `7` 加 `11`、`15`……
- 最倒霉的情况：`[2, 3, 4, 5]` target=9（2+3=5，2+4=6，2+5=7，3+4=7，3+5=8，4+5=9 ✓ 最后一轮才找到）

**分步演示（最坏情况）**：
```
i=0, j=1→2+3=5 ✗
i=0, j=2→2+4=6 ✗
i=0, j=3→2+5=7 ✗
i=1, j=2→3+4=7 ✗
i=1, j=3→3+5=8 ✗
i=2, j=3→4+5=9 ✓ 返回[2,3]
```

**致命伤是**：时间复杂度 **O(n²)** —— 数组有10000个数就要算约5000万次加法，CPU直接冒烟。

---

### 第4步：终极优化（核心亮点）

**本质比喻**：就一句话：**「边走边记，来一个人就问：我缺的那个伴儿你见过没？」**

**对比演示**（同一个例子 `nums=[2,7,11,15], target=9`）：

| 优化前（暴力法）             | 优化后（哈希表法）                           |
| ---------------------------- | -------------------------------------------- |
| 拿2和7加 → 找到了            | 看2，缺7，把「2在0号」记下来                 |
| （运气好一次结束）           | 看7，问哈希表「7在吗？」→ 在！0号！返回[0,1] |
| 但如果最坏情况要试遍所有组合 | **永远只遍历一遍数组，每个数只问一次哈希表** |

**精炼总结**：**每遍历一个数，只查一次哈希表看「target - 当前数」是否存在，而不是跟前面所有数挨个加一遍。**

### 第5步：C++代码实现

```cpp
#include <vector>
#include <unordered_map>
using namespace std;

class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        // 哈希表：key存数字，value存下标
        unordered_map<int, int> seen;

        for (int i = 0; i < nums.size(); i++) {
            int need = target - nums[i];  // 我需要找的搭档

            // ⭐ 为什么这么写：哈希表的find是O(1)查找，瞬间知道搭档之前来过没
            if (seen.find(need) != seen.end()) {
                // 找到了！返回搭档的下标和当前下标
                return {seen[need], i};
            }

            // ⭐ 为什么这么写：没找到搭档，就把自己存进哈希表，供后面的数来问
            seen[nums[i]] = i;
        }

        // 题目保证有解，但为了编译通过随便返回个空
        return {};
    }
};
```

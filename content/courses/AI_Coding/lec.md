# Vibe Coding 与 AI 编程实践

## 一、Vibe Coding 的利弊

### 定义
- 高度依赖 AI 生成代码，使用通用提示词
- 让 AI“写出整个功能”，用户只做少量修改

### 优点
| 优点     | 说明                       |
| -------- | -------------------------- |
| 快速启动 | 几分钟内生成可运行代码骨架 |
| 降低门槛 | 初学者也能快速看到成果     |
| 适合原型 | 快速验证想法               |

### 缺点
| 缺点         | 说明                       |
| ------------ | -------------------------- |
| 难以维护     | 代码结构混乱               |
| 调试困难     | bug 难以定位               |
| 学习缺失     | 跳过基础概念               |
| 提示词天花板 | 纯自然语言无法处理复杂逻辑 |

### 生产力曲线
- 初期：AI 快速产出
- 后期：边际效益递减，需人工介入

## 三、模型选择与提示策略

### 模型选择
| 模型              | 评价             |
| ----------------- | ---------------- |
| Deepseek / Claude | ✅ 问题最少       |
| ChatGPT 免费版    | ⚠️ 需更多调试     |
| Pro 模型          | ✅ 显著优于免费版 |

### 成功提示策略
| 策略         | 说明                     |
| ------------ | ------------------------ |
| 详细提示词   | 上下文 + 示例 + 具体需求 |
| 占位符先行   | 先结构，后内容           |
| 迭代开发     | 基础 → 功能1 → 功能2     |
| 生成代码片段 | 避免重写整个文件         |
| 避免模板     | 从现成模板开始反而困难   |

### 调试流程
```
生成片段 → 手动小改 → 无法实现则改变设计 → 换模型 → 寻求人类反馈
```

### 通用建议
- 不满足于首次结果
- 多模型尝试
- 重视上下文
- 留意意外变化
- 学会舍弃

## 四、提示词工程实验（俄罗斯方块）

### 实验设置
- 任务：Python + cmu_graphics 实现俄罗斯方块
- 时间：15 分钟
- 模型：Claude Sonnet 4.5

### 结果对比
| 策略           | 耗时     | 结果       |
| -------------- | -------- | ---------- |
| 简单提示       | ~2分45秒 | ✅ 完全可用 |
| 计划-调整-执行 | ~7分钟   | ⚠️ 速度慢   |
| 详细提示       | ~15分钟  | ❌ 需修复   |

### 结论
> 对定义良好的任务，**简单直接提示**最佳。当前 AI 能力边界 ≈ 《愤怒的小鸟》复杂度。

## 五、高质量提示词要点

### 要素
- 避免歧义
- 层次递进（目标 → 细节）
- 提供示例
- 请求澄清（“如有疑问请先提问”）
- 要求双检查 + 迭代

### 迭代提示示例
| 场景     | 提示                                            |
| -------- | ----------------------------------------------- |
| 功能缺失 | “This is an improvement, but we still need ___” |
| 视觉效果 | “Make it modern, but don't change game logic”   |
| 顽固 bug | “Don't hardcode rotations”                      |

### 角色设定
```
“你是简洁高效的代码助手，直接给出解决方案，避免空洞奉承。”
```

> ⚠️ 没有“万能超级提示词”，学会结构逻辑比照抄重要。

## 七、常见陷阱

| 陷阱         | 解决               |
| ------------ | ------------------ |
| 过度工程化   | 先试简单提示       |
| 需求模糊     | 提供示例和格式     |
| 调试信息缺失 | 要求 AI 添加 print |
| 一次要求太多 | MVP → 迭代         |
| 不迭代放弃   | 提供反馈让 AI 改进 |

## 十、代码模板：cmu_graphics 骨架

from cmu_graphics import *

def onAppStart(app):
    app.width = 800
    app.height = 600
    app.stepsPerSecond = 30
    app.player = {'x': 400, 'y': 550, 'size': 20}
    app.obstacles = []

def redrawAll(app):
    drawRect(0, 0, app.width, app.height, fill='lightGreen')
    drawCircle(app.player['x'], app.player['y'], app.player['size'], fill='green')
    for obs in app.obstacles:
        drawRect(obs['x'], obs['y'], 30, 30, fill='red')

def onKeyPress(app, key):
    if key == 'left':  app.player['x'] -= 20
    if key == 'right': app.player['x'] += 20
    if key == 'up':    app.player['y'] -= 20
    if key == 'down':  app.player['y'] += 20

def onStep(app):
    pass  # 移动障碍物、碰撞检测等

def main():
    runApp()

if __name__ == '__main__':
    main()

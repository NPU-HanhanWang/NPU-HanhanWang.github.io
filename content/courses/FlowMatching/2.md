# 第二节：快速入门和关键概念 (Section 2: Quick tour and key concepts)

鉴于我们可以访问从某个目标分布 $q$（定义在 $\mathbb{R}^d$ 上）中抽取的训练数据集，我们的目标是构建一个能够从 $q$ 中生成新样本的模型。

为了解决这个任务，流匹配（FM）构建了一条概率路径 $(p_t)_{0 \leq t \leq 1}$，从已知的源分布 $p_0 = p$ 到数据目标分布 $p_1 = q$，其中每个 $p_t$ 都是 $\mathbb{R}^d$ 上的一个分布。

具体来说，FM 是一个简单的回归目标，用于训练速度场神经网络，该网络描述样本的瞬时速度——之后用于沿着概率路径 $p_t$ 将源分布 $p_0$ 转换为目标分布 $p_1$。

训练完成后，我们通过以下方式从目标分布中生成一个新样本 $X_1 \sim q$：（i）从源分布中抽取一个新样本 $X_0 \sim p$，以及（ii）求解由该速度场确定的常微分方程（ODE）。

更正式地说，一个 ODE 由一个依赖于时间的向量场 $u:[0,1]\times \mathbb{R}^d\to \mathbb{R}^d$ 定义，在我们的情况下，该向量场是用神经网络建模的速度场。

这个速度场确定了一个依赖于时间的流 $\psi :[0,1]\times \mathbb{R}^d\to \mathbb{R}^d$，定义为

\[\frac{\mathrm{d}}{\mathrm{d}t}\psi_t(x) = u_t(\psi_t(x)),\]

其中 $\psi_t \coloneqq \psi (t,x)$ 且 $\psi_0(x) = x$。

如果速度场 $u_t$ 的流 $\psi_t$ 满足

\[X_{t}\coloneqq \psi_{t}(X_{0})\sim p_{t}\mathrm{~for~}X_{0}\sim p_{0}. \quad (2.1)\]

则称该速度场 $u_t$ 生成概率路径 $p_t$。

根据上面的方程，速度场 $u_t$ 是通过求解上述 ODE 从 $p_t$ 中采样的唯一工具。

如图 2d 所示，将 ODE 求解到 $t = 1$ 为我们提供了样本 $X_1 = \psi_1(X_0)$，这些样本类似于目标分布 $q$。

因此，总而言之，流匹配的目标是学习一个向量场 $u_t^\theta$，使得其流 $\psi_t$ 生成一条满足 $p_0 = p$ 和 $p_1 = q$ 的概率路径 $p_t$。

**图2** 流匹配蓝图。(a) 目标是找到一个流，将来自已知源分布或噪声分布 $p$ 的样本 $X_0$ 映射到来自未知目标分布或数据分布 $q$ 的样本 $X_1$。(b) 为此，设计一条时间连续的概率路径 $(p_t)_{0 \leq t \leq 1}$，在 $p := p_0$ 和 $q := p_1$ 之间插值。(c) 在训练期间，使用回归来估计已知生成 $p_t$ 的速度场 $u_t$。(d) 为了抽取一个新的目标样本 $X_1 \sim q$，从 $t = 0$ 到 $t = 1$ 积分估计的速度场 $u_t^\theta(X_t)$，其中 $X_0 \sim p$ 是一个新的源样本。

使用上述符号，流匹配的目标是学习参数 $\theta$，这些参数属于用神经网络实现的速度场 $u_t^\theta$。

正如引言中所述，我们通过两个步骤来实现这一点：设计一条在 $p$ 和 $q$ 之间插值的概率路径 $p_t$（见图 2b），并通过回归训练一个生成 $p_t$ 的速度场 $u_t^\theta$（见图 2c）。

因此，让我们继续执行流程的第一步：设计概率路径 $p_t$。

在这个例子中，令源分布 $p := p_0 = \mathcal{N}(x|0, I)$，并将概率路径 $p_t$ 构建为条件概率路径 $p_{t|1}(x|x_1)$ 的聚合，每个条件路径都依赖于训练数据集中的一个数据样本 $X_1 = x_1$。（图 3a 展示了其中一个条件路径。）

因此，概率路径 $p_t$ 遵循以下表达式：

\[p_t(x) = \int p_{t|1}(x|x_1)q(x_1)\mathrm{d}x_1,\mathrm{~where~}p_{t|1}(x|x_1) = \mathcal{N}(x|x_1,(1 - t)^2 I). \quad (2.2)\]

这条路径，也称为条件最优传输或线性路径，拥有一些我们将在本文后续研究的理想特性。

使用这个概率路径，我们可以通过抽取 $X_0 \sim p$，抽取 $X_1 \sim q$，并取它们的线性组合来定义随机变量 $X_t \sim p_t$：

\[X_{t} = tX_{1} + (1 - t)X_{0}\sim p_{t}. \quad (2.3)\]

我们现在继续执行流匹配流程的第二步：将我们的速度场 $u_t^\theta$（通常用神经网络实现）回归到一个已知能生成期望概率路径 $p_t$ 的目标速度场 $u_t$。

为此，流匹配损失定义为：

\[\mathcal{L}_{\mathrm{FM}}(\theta) = \mathbb{E}_{t,X_t}\left\| u_t^\theta (X_t) - u_t(X_t)\right\| ^2,\mathrm{~where~}t\sim \mathcal{U}[0,1]\mathrm{~and~}X_t\sim p_t. \quad (2.4)\]

在实践中，人们很少能直接实现上述目标，因为 $u_t$ 是一个控制两个高维分布之间联合变换的复杂对象。

幸运的是，通过将损失条件化在从训练集中随机抽取的单个目标样本 $X_1 = x_1$ 上，目标函数可以大大简化。

为了看到这一点，借用方程 (2.3) 来实现条件随机变量

\[X_{t|1} = tx_1 + (1 - t)X_0\sim p_{t|1}(\cdot |x_1) = \mathcal{N}(\cdot |tx_1,(1 - t)^2 I). \quad (2.5)\]

使用这些变量，求解 $\frac{\mathrm{d}}{\mathrm{d}t} X_{t|1} = u_t(X_{t|1}|x_1)$ 得到条件速度场

\[u_t(x|x_1) = \frac{x_1 - x}{1 - t}, \quad (2.6)\]

该速度场生成条件概率路径 $p_{t|1}(\cdot |x_1)$。（关于这两个条件对象的图示，见图 3c。）

有了用于生成所设计条件概率路径的条件速度场的简单方程 (2.6)，我们可以构建 (2.4) 中流匹配损失的一个可处理版本。

这就是条件流匹配（CFM）损失：

\[\mathcal{L}_{\mathrm{CFM}}(\theta) = \mathbb{E}_{t,X_t,X_1}\| u_t^\theta (X_t) - u_t(X_t|X_1)\| ^2,\mathrm{~where~}t\sim U[0,1],X_0\sim p,X_1\sim q, \quad (2.7)\]

其中 $X_t = tX_1 + (1 - t)X_0$。

在代码 1 中，我们展示了上述条件流匹配损失的简单 PyTorch 实现。该代码还使用中点 ODE 求解器（一种改进的欧拉方法）实现了采样过程。

在训练循环之后，我们展示了生成的样本，这些样本说明了模型成功地将源分布（高斯分布）转换为双月形分布。

**代码 1：带有 CFM 损失的双月数据集上的流匹配训练**

```python
import torch
from torch import nn, Tensor
import matplotlib.pyplot as plt
from sklearn.datasets import make_moons

class Flow(nn.Module):
    def __init__(self, dim: int = 2, h: int = 64):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(dim + 1, h), nn.ELU(),
            nn.Linear(h, h), nn.ELU(),
            nn.Linear(h, h), nn.ELU(),
            nn.Linear(h, dim))

    def forward(self, x_t: Tensor, t: Tensor) -> Tensor:
        return self.net(torch.cat((t, x_t), - 1))

    def step(self, x_t: Tensor, t_start: Tensor, t_end: Tensor) -> Tensor:
        t_start = t_start.view(1, 1).expand(x_t.shape[0], 1)
        # 为简化，本示例使用中点 ODE 求解器
        return x_t + (t_end - t_start) * self(x_t + self(x_t, t_start) * (t_end - t_start) / 2,
                        t_start + (t_end - t_start) / 2)

# 训练
flow = Flow()
optimizer = torch.optim.Adam(flow.parameters(), 1e- 2)
loss_fn = nn.MSELoss()

for _ in range(10000):
    x_1 = Tensor(make_moons(256, noise=0.05)[0])
    x_0 = torch.randn_like(x_1)
    t = torch.randn(len(x_1), 1)
    x_t = (1 - t) * x_0 + t * x_1
    dx_t = x_1 - x_0
    optimizer.zero_grad()
    loss_fn(flow(x_t, t), dx_t).backward()
    optimizer.step()

# 采样
x = torch.randn(300, 2)
n_steps = 8
fig, axes = plt.subplots(1, n_steps + 1, figsize=(30, 4), sharex=True, sharey=True)
time_steps = torch.linspace(0, 1.0, n_steps + 1)

axes[0].scatter(x.detach()[0, 0], x.detach()[1, 1], s=10)
axes[0].set_title(f't = {time_steps[0]:.2f}')
axes[0].set_xlim(-3.0, 3.0)
axes[0].set_ylim(-3.0, 3.0)

for i in range(n_steps):
    x = flow.step(x, time_steps[i], time_steps[i + 1])
    axes[i + 1].scatter(x.detach()[0, 0], x.detach()[1, 1], s=10)
    axes[i + 1].set_title(f't = {time_steps[i + 1]:.2f}')

plt.tight_layout()
plt.show()
